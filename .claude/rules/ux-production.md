# UX & 프로덕션 기준 규칙

## 프로덕션 환경 기준
- 앱 개선/변경 제안 시 항상 **온라인 배포 환경**(Vercel + 실제 유저)에서 작동하는 기준으로 판단
- localhost 전용 workaround 금지. 환경변수, HTTPS, CORS 등 프로덕션 차이를 항상 고려
- 배포 시 깨질 수 있는 변경은 사전에 명시

## 유저 편의 검증 필수
- 코드 변경이 UX에 영향을 줄 때 → 유저 관점에서 이해 가능한지 검증
- `docs/UX_PRINCIPLES.md`의 원칙을 참조하여 판단
- 핵심 체크: 3-click rule, Hick's Law(선택지 최소화), 즉각 피드백, 에러 상태 처리

## UI 상태 5종 필수
모든 화면에 다음 5가지 상태가 존재하는지 확인:
1. Empty (빈 상태) — 첫 사용 가이드
2. Loading (로딩) — 스켈레톤 또는 스피너
3. Partial (일부 데이터)
4. Error (에러) — 복구 액션 제공
5. Ideal (정상)
