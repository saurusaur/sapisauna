# 오류 해결 프로세스

## Micro-fix 금지
오류 발생 시 해당 에러만 좁게 보고 즉석 패치하지 않는다.
반드시 전체 흐름과 해당 기능의 맥락을 먼저 파악한 뒤 수정한다.

## 3사이클 검토법
1. **제안** — 가능한 해결 후보를 도출
2. **분석** — 각 후보의 영향 범위, 안정성, 효율성 평가
3. **비판** — 약점과 부작용 탐색

이 사이클을 3회 반복한 뒤, 후보를 최종 재검토하여 최적 솔루션을 선택한다.

## Decision Point는 사용자에게
해결 과정에서 발생하는 설계 결정(방향, 트레이드오프)은 임의로 정하지 말고 사용자에게 질문하여 확인받는다.
