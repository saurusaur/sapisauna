## 날카로운 관찰

**감사(audit)가 핵심 구멍을 찾아냈다.** bath_gender 분리 작업 이후 "혹시 다른 것도 빠진 거 있나?" 한 마디가 CRITICAL 4건을 터뜨렸다. 가장 큰 건 logs INSERT 로직 자체가 없다는 것 — UI에서 열심히 입력받지만 Supabase에 도달하는 코드가 프로젝트에 없었다. 이건 감사 안 했으면 배포 후에 발견했을 문제다.

**중복 로직이 이미 상당히 쌓여있다.** 즐겨찾기 유틸 3곳 복붙, getFacilityLabel 4곳, ChipSelect 2곳, TRIBE_COLORS 3곳. 기능 추가에 급급해서 공통화를 미루다 보니 복사-붙여넣기 부채가 12건까지 쌓였다. 아직 프로젝트 초기라 지금 잡으면 싸게 잡을 수 있다.

**커밋 위생이 2/28 회고보다 후퇴했다.** 지난번에 "커밋 분리 전략의 자기검증"이 잘됐다고 했는데, 오늘은 DB 연동 + bath_gender 분리 + 문서 정리를 하나의 커밋에 모았다. 31파일 변경은 리뷰 불가능한 크기. 작업 중간에 커밋을 찍는 습관이 아직 정착되지 않았다.

## 잘 된 것

**"이 시설은 어떤 종류" vs "오늘 나는 어디 이용"의 도메인 분리.** bath_gender를 장소 속성과 로그 기록으로 나눈 건 데이터 모델 측면에서 올바른 판단이었다. 장소는 `male-only/female-only`, 로그는 `male/female` — 의미가 실제로 다르고, 나중에 필터/통계를 만들 때 혼동을 방지한다.

**감사 → 문서화 → 백로그 등록 플로우.** 문제를 발견하고 바로 코드로 뛰어들지 않고, AUDIT 문서 → PLAN 문서 → BACKLOG P0/P1 등록으로 체계적으로 정리한 뒤 "하나하나 검토하면서 진행"하기로 한 결정. 급하게 고치다가 더 큰 걸 놓치는 패턴을 피했다.

**세신(scrub) 카테고리 재분류.** 사용자가 "편의시설이 아니라 추가시설"이라고 지적했을 때 바로 수정. AMENITY_LABEL_MAP 중복 추가 실수도 사용자 질문 한 마디에 바로 잡힘 — 사용자 피드백 루프가 빠르게 동작하고 있다.
