# Naver Maps 검색어 포맷 최적화 테스트

**날짜**: 2026-03-24
**목적**: 네이버 지도 검색 링크에서 검색어를 어떤 형식으로 구성해야 정확한 결과가 나오는지 비교 테스트
**테스트 방법**: Naver Local Search API (`openapi.naver.com/v1/search/local.json`)로 30개 장소 × 6개 포맷 검색
**데이터 소스**: Supabase `places` + `place_sources` (source=naver) — KR 시설 30곳

## 4. 실패 케이스 심층 분석

### Case 1: "하남사우나" — 이름이 지역명과 겹침
- **A (name only)**: `하남사우나` → 경기도 하남시 "주심유황참숯가마" 반환 (하남=지역명으로 해석)
- **E (name + 서울)**: `하남사우나 서울` → 정확히 서울 용산구 하남사우나 반환
- **교훈**: 이름에 지역명이 포함된 경우, 도시명 추가가 필수

### Case 2: "강남목욕탕" — 이름이 다른 지역 연상
- **A (name only)**: `강남목욕탕` → 서울 강남구 다른 시설 반환 (강남=강남구로 해석)
- **E (name + 천안)**: `강남목욕탕 천안` → 정확히 충남 천안시 강남목욕탕 반환
- **교훈**: 흔한 지역명이 상호에 포함된 경우, 실제 소재 도시명 추가가 필수

### Case 3: "현대그린사우나" — Naver에 미등록
- 경기도 광주시 소재 시설이 Naver Local에 등록되어 있지 않음
- `현대그린사우나` (name only)로 검색하면 서울 관악구 동명 시설이 반환됨
- **교훈**: 이건 검색 포맷 문제가 아니라 데이터 품질 이슈. 어떤 포맷을 써도 해결 불가

### Case 4: "온유재스파마사지 상무점" + "서구" — 흔한 구 이름
- **D/F (+서구)**: `온유재스파마사지 상무점 서구` → 검색 결과 없음
- **E (+광주)**: `온유재스파마사지 상무점 광주` → 정확 매칭
- **교훈**: "서구", "중구", "동구" 같은 흔한 구 이름만 붙이면 오히려 검색 정확도 하락

### Case 5: "킹사우나찜질방" + 시도 약어
- **C (+경상남)**: `킹사우나찜질방 경상남` → 검색 결과 없음
- **E (+창원)**: `킹사우나찜질방 창원` → 정확 매칭
- **교훈**: 시도 약어(경상남, 충청북 등)는 자연스러운 검색어가 아님. 통용 도시명이 훨씬 유효

## 5. 추천

### Best Format: **E (name + 통용 도시명)**

```
https://map.naver.com/v5/search/${encodeURIComponent(name + ' ' + commonCityName)}?c=${lng},${lat},17
```

**이유**:
1. **97% 정확도** — 30건 중 29건 매칭 (유일한 실패는 Naver 미등록 시설)
2. **URL 가독성** — `하남사우나+서울`이 `하남사우나+서울특별시+용산구`보다 자연스러움
3. **핵심 오매칭 해결** — Format A의 지역명 혼동 문제(하남사우나→하남시, 강남목욕탕→강남구)를 해결
4. **짧은 쿼리** — 검색 엔진이 짧고 핵심적인 쿼리에서 더 잘 작동

### 통용 도시명 매핑 규칙

```typescript
function getCommonCityName(address: string): string {
  const cityMap: Record<string, string> = {
    '서울특별시': '서울',
    '부산광역시': '부산',
    '대구광역시': '대구',
    '인천광역시': '인천',
    '광주광역시': '광주',
    '대전광역시': '대전',
    '울산광역시': '울산',
    '세종특별자치시': '세종',
    '제주특별자치도': '제주',
  };

  const parts = address.split(' ');
  const sido = parts[0];

  // 특별시/광역시/특별자치 → 매핑 테이블
  if (cityMap[sido]) return cityMap[sido];

  // 도 단위 (경기도 수원시 → 수원) — 두 번째 토큰에서 '시' 제거
  if (sido.endsWith('도') && parts.length > 1) {
    return parts[1].replace('시', '');
  }

  return sido;
}
```

### Fallback: Format B (현 구현) — 보수적 선택지

Format B (`name + 시도 시군구`)는 테스트 25건 전원 100% 매칭. 정확도 면에서 가장 안전하나:
- URL이 길어짐
- "서울특별시 용산구" 같은 공식 표기가 자연스러운 검색어는 아님
- 실사용에서 Naver가 이를 잘 파싱하지만, 불필요한 중복이 있음

### 권장 하이브리드 전략

```
1차: Format E (name + 통용 도시명) — 기본
2차: 검색 결과 0건이면 Format B (name + 시도 시군구) 로 재시도
```

단, 네이버 **지도 URL** 검색은 API 검색과 동작이 다를 수 있으므로, 실제 `map.naver.com/v5/search/` URL에서도 E 포맷이 잘 작동하는지 브라우저 수동 확인 권장.
