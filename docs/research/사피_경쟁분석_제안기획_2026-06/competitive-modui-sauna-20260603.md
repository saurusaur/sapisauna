## 1. 한 줄 정의

**"사우나판 카카오맵/구글맵."** 전국 사우나·목욕탕·찜질방을 지도에 뿌리고, 핀을 눌러
주소·전화·영업시간·리뷰를 보고, 원탭 체크인으로 뱃지를 모으는 **위치기반 디렉토리 앱**.

- 포지셔닝: **지도 우선(map-first) 발견 도구** (사-피는 기록·큐레이션 우선)
- 성격: 넓고 얕음 (사-피는 좁고 깊음)

## 2. 기능 인벤토리 (실측)

| 영역 | 기능 | 메모 |
|---|---|---|
| **지도 탭** | 핀·필터·검색·내위치·현위치 재검색·거리순 하단 카드 | 표준 지도앱 패턴, 완성도 높음 |
| **필터** | 종류(목욕탕/찜질방/한증막)·시도(전국 17)·사우나실 유무·정렬(가까운/평점/리뷰순) | 시설 속성 필터는 "사우나실 있음" 단 1개 |
| **상세** | 주소·전화·영업시간·즐겨찾기·길찾기·카톡공유·리뷰(별점+텍스트+닉네임)·체크인 | 영업시간은 카카오맵/네이버 **링크 위임**(자체관리 X) |
| **체크인** | **그냥 누르면 체크인**(원탭, 무마찰) → 방문기록+뱃지 | 위치인증·기록입력 없음 = 발도장 수준 |
| **뱃지** | 6종: 첫발걸음(1)·단골(5)·마니아(10)·마스터(30)·찜질방탐험가(5)·한증막고수(3) | **전부 "방문 횟수" 기반** |
| **랭킹 탭** | 최다조회·최다공유·최고평점·최다리뷰 (각 Top30) | 시스템 자동집계(유저 큐레이션 아님) |
| **마이 탭** | 자동닉네임(사우나테마)·방문수·뱃지수·방문기록 | |
| **하단 네비** | 지도 · 랭킹 · 즐겨찾기 · 문의 · 마이 | |

## 3. 데이터 현황 (중요)

- **등록 업소 5,576개** — 사-피(~230)의 약 24배. *겉보기 압도적 커버리지.*
- **그러나 품질 노이즈 큼**: 사업자등록/공공데이터 스크랩으로 추정.
  실측 리스트에 `짐구공 헬스 PT 골프`, `버핏그라운드`, `(주)피트비엠`,
  `더블에스 휘트니스`, 호텔 피트니스 등 **헬스장·PT·골프장이 "목욕탕"으로 다수 혼입.**
- **리뷰 거의 0** ("리뷰 없음" 대부분) → **콜드스타트, 커뮤니티 밀도 사실상 0.**
- 시설 메타는 종류 1개 + 🔥(사우나실 유무) 플래그뿐. 온도·노천·냉탕·세신·타투 등 **사우나 전문 속성 전무.**

## 4. 사-피 대비 강·약점

### 모두의 사우나가 앞선 것 (= 사-피가 좁혀야 할 테이블스테이크)
1. **커버리지 양** — "검색하면 일단 나온다"는 인상. 발견 단계 진입장벽 낮음.
2. **지도 우선 UX** — 위치기반 발견이 메인. 사-피는 탐색이 리스트 중심(지도뷰 *로드맵 단계*) → **추격 시급.**
3. **무마찰 원탭 체크인** — 누르면 끝. 진입 비용 0.
4. **카카오맵/네이버 위임** — 영업시간 최신성을 자체관리 안 하고 외부에 위임(운영비↓).
5. **카카오톡 네이티브 공유** — 한국 바이럴 핵심 채널 선점.

### 객관적으로 유효한 사-피 차별점 — 단 2개

> ⚠️ 1차 작성 시 사-피에 후했음. 냉정하게 보면 지금 단계에서 **실질적 우위는 아래 2개뿐.**
> 나머지(기록 깊이·큐레이션·칭호)는 *미검증·복제 가능·네트워크효과 미발생*으로 현 단계 강점 아님.

1. **데이터 노이즈 (그들의 약점)** — 스크랩 기반이라 헬스장·PT·골프장이 "목욕탕"으로 혼입.
   사-피가 *큐레이션된 정확한 사우나 DB*를 유지하면 정면 차별화. **단, 조건부**: 사-피 230건은
   그들 5,576건 대비 양이 절대 열세 → "정확하지만 안 나옴 vs 시끄럽지만 다 나옴"에서
   발견 유틸리티는 흔히 *양*이 이긴다. 노이즈 우위는 **커버리지를 어느 선까지 끌어올린 뒤에야** 성립.
2. **단일 페르소나 (그들의 약점)** — 목욕/사우나/찜질을 한 덩어리로 봄. 사-피의 **3-트라이브**는
   한국 사우나 문화에 맞는 개념적 차별화로 유효. **단, 유저가 트라이브 구분을 실제로 원하는지는 미검증**
   (대다수가 그냥 목욕파일 가능성 — 베타 실측 필요).

### 강점으로 보였지만, 현 단계엔 아닌 것 (정직하게)

- **기록 깊이(숏/딥로그)** — 기능은 있으나 *수요 미검증·복제 가능*. 경쟁사 리뷰란이 거의 비어 있다는 건
  애초에 다수 유저가 깊은 기록을 안 한다는 신호일 수 있음. 딥로그가 오버엔지니어링일 위험.
- **SA-LIST 큐레이션·구독** — 네트워크 효과 기능. 유저 없는 상태에선 *빈 껍데기*(empty state)라 가치 0.
  콜드스타트에선 강점이 아니라 부채에 가까움.
- **칭호·정체성** — 경쟁사도 뱃지 보유. 340 vs 6은 *아무도 안 모으면 무의미*. 현 단계 실질 강점 아님(유저 지적).
- **콜드스타트는 양쪽 공통** — 경쟁사 리뷰 0을 약점으로 적었지만, 사-피도 클로즈드 알파(230 시드)로 같은 처지.
  이건 사-피의 우위가 아님.

## 5. 전략적 시사점 (기능 우선순위 영향)

**따라가면 안 되는 것 (함정)**
- 5,576 커버리지 경쟁. 스크랩 노이즈 게임은 사-피 정체성("정확·전문")을 훼손. → **질로 승부.**

**반드시 좁혀야 할 갭 (모두의 사우나가 선점한 기본기) — 우선순위↑**
- ① **탐색 지도뷰 + 내주변/geolocation** (이미 로드맵 → **상향 조정 권고**)
- ② **숏로그를 원탭 체크인급으로 더 빠르게** (무마찰 진입 모방)
- ③ **카카오톡 공유** (현재 인스타 스토리 위주 → 카톡 채널 보강)

**검증해야 할 베팅 (강점이라 단정 말고 베타로 확인)**
- **데이터 큐레이션 우위**: 노이즈 차별화는 *커버리지를 일정 임계치까지 올린 뒤에야* 성립 → "정확한 사우나만"을
  무기로 쓰려면 시드 확장이 전제. 양 열세인 채로는 우위가 안 됨.
- **3-트라이브**: 개념은 차별적이나 *유저가 실제로 원하는지 미검증* → 온보딩 트라이브 선택률·트라이브별 행동 차이를
  베타 1순위 측정 지표로.
- **깊은 기록·큐레이션·칭호는 "베팅"이지 "해자"가 아님** — 수요·네트워크효과·참여가 붙어야 가치 발생. 콜드 상태에선 0.

**한 줄 결론 (수정)**
> 모두의 사우나는 **넓이(커버리지·지도·무마찰)**로 먼저 치고 나갔다.
> 사-피가 지금 객관적으로 가진 카드는 **①데이터 정확성 ②3-트라이브 개념** 둘뿐이며, 둘 다 *조건부·미검증*이다.
> 나머지(딥로그·SA-LIST·칭호)는 잠재적 베팅일 뿐 현 단계 우위가 아니다.
> **가장 위험한 시나리오**: 경쟁사가 노이즈를 정리하고 데이터 밀도/리뷰를 먼저 쌓는 것 → 사-피의 유일한 두 카드마저 잠식.
> 444 협업 시드로 **콜드스타트 + 데이터 양 격차를 빨리 좁히는 게 시간 싸움.**
