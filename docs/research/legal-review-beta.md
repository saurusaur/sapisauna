# 사피(SA-PI) 베타 출시 법률 검토

> 작성일: 2026-03-20
> 대상: 한국 서비스 PWA (웹앱), Google OAuth 로그인, Supabase 백엔드
> 목적: 베타 테스트 전 개인정보보호법(PIPA) 및 관련 법령 준수 여부 점검

## 1. 수집하는 개인정보 목록

### 1-1. 인증 단계 (Google OAuth → Supabase Auth)

| 항목 | 수집 방식 | 필수/선택 | 비고 |
|------|-----------|-----------|------|
| Google 이메일 | OAuth 자동 | 필수 | Supabase `auth.users`에 저장 |
| Google UID | OAuth 자동 | 필수 | 내부 식별자 |
| Google 프로필명 | OAuth 자동 | 필수 | Supabase 메타데이터 (앱에서 미사용) |
| Google 프로필 사진 URL | OAuth 자동 | 필수 | Supabase 메타데이터 (앱에서 미사용) |

### 1-2. 온보딩 (사용자 직접 입력)

| 항목 | DB 컬럼 | 필수/선택 | 비고 |
|------|---------|-----------|------|
| 닉네임 | `users.nickname` | 필수 | 영문 대문자+숫자+언더스코어, 2-10자 |
| 성별 | `users.gender` | **선택** | male/female, "말하고 싶지 않아요" 가능 |
| 사용자 타입 | `users.user_types` | 필수 | bather/saunner/jimi 복수선택 |
| 최선호 타입 | `users.primary_type` | 필수 | 기능 맞춤에 사용 |

### 1-3. 사우나 기록 (Quick Log)

| 항목 | 민감도 | 비고 |
|------|--------|------|
| 방문 장소 (place_id) | 낮음 | 공개 장소 정보 |
| 방문 날짜/시간 | 중간 | 행동 패턴 추론 가능 |
| 탕 구분 (bath_gender) | 중간 | 성별 간접 추론 가능 |
| 온도/점수 등 루틴 데이터 | 낮음 | 기능적 데이터 |
| 재방문 의향 점수 | 낮음 | |

### 1-4. 딥로그 (Deep Log)

| 항목 | 민감도 | 비고 |
|------|--------|------|
| 동행자 유형 | 중간 | alone/friend/family/partner |
| 비용 | 중간 | 소비 패턴 |
| 혼잡도 | 낮음 | |
| 메모 (자유 텍스트) | **높음** | 사용자가 민감 정보 입력 가능 |
| 세신/매점 이용 여부 및 만족도 | 낮음 | |

### 1-5. 장소 등록

| 항목 | 민감도 | 비고 |
|------|--------|------|
| 장소명, 주소 | 낮음 | 공개 정보 |
| 위도/경도 | 낮음 | 네이버/구글 API에서 가져온 **장소 좌표** (사용자 위치 아님) |
| 시설 스펙 | 낮음 | 공개 정보 성격 |
| 등록자 ID (created_by) | 낮음 | 기여 추적용 |

### 1-6. 수집하지 않는 정보 (확인됨)

- **사용자 위치(GPS)**: 앱이 기기 위치 권한을 요청하지 않음. 장소 좌표는 검색 API에서 가져옴
- **사진/이미지**: Supabase Storage 미사용. 이미지는 클라이언트 로컬에서만 처리
- **연락처/전화번호**: 수집하지 않음
- **결제 정보**: 수집하지 않음
- **쿠키 기반 추적**: Supabase 세션 쿠키만 사용 (마케팅 추적 없음)

## 2. 한국 개인정보보호법(PIPA) 기준 필수 조치사항

### 2-1. 개인정보 처리방침 ✅ 필수

**근거**: 개인정보보호법 제30조 — 개인정보처리자는 개인정보 처리방침을 수립하고 공개해야 함.
소규모 베타라도 **1명 이상의 외부 사용자 데이터를 처리하면 적용 대상**임.

**필수 포함 내용** (법 제30조 제1항):

| # | 항목 | 사피 해당 내용 |
|---|------|---------------|
| 1 | 처리 목적 | 사우나 기록, 공유, 추천 서비스 제공 |
| 2 | 처리 항목 및 보유기간 | 이메일, 닉네임, 성별(선택), 기록 데이터 / 회원 탈퇴 시까지 |
| 3 | 제3자 제공 사항 | 없음 (단, 위탁 처리 별도 기재) |
| 4 | 처리 위탁 사항 | Supabase, Vercel, Google, Naver (검색 API) |
| 5 | 정보주체 권리 및 행사 방법 | 열람, 정정, 삭제, 처리정지 요구 방법 |
| 6 | 파기 절차 및 방법 | 탈퇴 시 DB 삭제 (CASCADE) |
| 7 | 안전성 확보 조치 | RLS, HTTPS, OAuth 등 |
| 8 | 개인정보 보호책임자 | 이름, 연락처 (대표 또는 담당자) |
| 9 | 처리방침 변경 시 고지 방법 | 앱 내 공지 또는 설정 페이지 |

### 2-2. 이용약관 ⚠️ 권장 (법적 의무는 아니나 사실상 필수)

- 전자상거래법상 의무는 아니지만 (무료 서비스), **UGC(사용자 생성 콘텐츠)를 다루므로** 분쟁 방지를 위해 사실상 필수
- 장소 등록 데이터의 저작권/소유권 명시 필요
- 면책 조항 (베타 서비스 불안정성)

### 2-3. 동의 수집 방식 ✅ 필수

**현재 상태**: 로그인 페이지에 약관/개인정보 동의 절차 **없음**. 설정 페이지에 링크 버튼만 있으나 동작하지 않음 (onClick 핸들러 없음).

**필요 조치**:

| 조치 | 레벨 | 설명 |
|------|------|------|
| 로그인 시 필수 동의 체크박스 | ✅ 필수 | "이용약관 동의" + "개인정보 수집·이용 동의" |
| 선택 동의 분리 | ✅ 필수 | 성별 수집은 선택 항목 → 별도 동의 또는 온보딩에서 명시적 고지 |
| 만 14세 미만 확인 | ⚠️ 권장 | PIPA 제22조의2 — 법정대리인 동의 필요. 베타 단계에서는 약관에 "14세 이상" 명시로 대응 |
| 동의 기록 저장 | ✅ 필수 | `users` 테이블에 `terms_agreed_at`, `privacy_agreed_at` 컬럼 추가 권장 |

## 3. 위치정보법 적용 여부

### 결론: ❌ 적용되지 않음

| 검토 항목 | 결과 |
|-----------|------|
| 사용자 기기 GPS 수집 | 없음 — 위치 권한 요청 코드 없음 |
| IP 기반 위치 추적 | 없음 |
| 장소 좌표 저장 | 있음 — 그러나 **시설의 공개 주소/좌표**이지 사용자 위치가 아님 |

**위치정보의 보호 및 이용 등에 관한 법률**은 "개인위치정보" (특정 개인의 위치)를 대상으로 함.
사피는 **사용자 위치를 수집하지 않고**, 네이버/구글 API에서 가져온 **시설 좌표**만 저장하므로 위치정보법 대상이 아님.

> 추후 "내 근처 사우나 찾기" 같은 GPS 기반 기능 추가 시 재검토 필요.

## 4. 제3자 제공 및 처리 위탁

PIPA에서는 "제3자 제공"과 "처리 위탁"을 구분함.
사피의 경우 **제3자 제공은 없고**, 인프라 사업자에게 **처리를 위탁**하는 구조임.

### 4-1. 처리 위탁 현황

| 위탁 업체 | 위탁 업무 | 데이터 범위 | 소재국 |
|-----------|-----------|-------------|--------|
| **Supabase Inc.** | DB 호스팅, 인증, 세션 관리 | 전체 개인정보 | 미국 (AWS us-east-1 등) |
| **Vercel Inc.** | 웹앱 호스팅, 서버리스 함수 | 요청 로그, API 호출 데이터 | 미국/글로벌 CDN |
| **Google LLC** | OAuth 인증 | 이메일, 프로필 | 미국 |
| **Naver Corp.** | 장소 검색 API | 검색 쿼리 (개인정보 아님) | 한국 |

### 4-2. 국외 이전 고지 ✅ 필수

PIPA 제28조의8(2023 개정) — 개인정보의 국외 이전 시 정보주체에게 고지 필요.

**필수 고지 항목**:
- 이전되는 국가 (미국)
- 이전 일시 및 방법 (서비스 이용 시 실시간, 네트워크 전송)
- 이전받는 자 (Supabase, Vercel, Google)
- 이전 목적 및 보유기간

→ 개인정보 처리방침에 국외 이전 섹션 포함 필수.

| 개인정보 열람/삭제 요청 대응 | ✅ 필수 | 수동 대응이라도 절차 마련 필요 |

## 7. 개인정보 처리방침 초안 구조

```
1. 개인정보의 처리 목적
2. 수집하는 개인정보 항목
   - 필수: 이메일, 닉네임, 사용자 타입
   - 선택: 성별
   - 자동 수집: 서비스 이용 기록, 접속 로그
3. 개인정보의 처리 및 보유기간
   - 회원 탈퇴 시 즉시 파기 (단, 법령에 따른 보존 의무 제외)
4. 개인정보의 제3자 제공
   - 해당 없음
5. 개인정보 처리의 위탁
   - Supabase Inc. (DB/인증), Vercel Inc. (호스팅), Google LLC (OAuth)
6. 개인정보의 국외 이전
   - 미국 (Supabase, Vercel, Google) — 목적, 항목, 보유기간 명시
7. 정보주체의 권리·의무 및 행사 방법
   - 열람, 정정, 삭제, 처리정지 요구 방법
   - 설정 > 회원 탈퇴로 직접 삭제 가능
8. 개인정보의 파기 절차 및 방법
   - 전자적 파일: 복구 불가능한 방법으로 삭제
9. 개인정보의 안전성 확보 조치
   - HTTPS 암호화, Row Level Security, OAuth 2.0 인증
10. 개인정보 보호책임자
    - 성명, 직위, 연락처 (이메일)
11. 처리방침 변경에 관한 사항
    - 변경 시 앱 내 공지 및 설정 페이지 게시
12. 시행일: YYYY년 MM월 DD일
```

## 8. 이용약관 초안 구조

```
1. 목적
2. 정의 (서비스, 회원, 콘텐츠 등)
3. 서비스의 제공 및 변경
   - 베타 서비스 특성 고지 (불안정성, 데이터 손실 가능)
4. 회원가입 및 이용자격
   - 만 14세 이상
5. 회원의 의무
   - 허위 정보 등록 금지, 타인 명예 훼손 금지
6. 서비스 이용 제한 및 중지
7. 사용자 생성 콘텐츠(UGC) 권리
   - 장소 등록 데이터: 서비스 내 활용 허락 (비독점 라이선스)
   - 기록 데이터: 본인 소유, 통계 목적 익명 활용
8. 면책 조항
   - 베타 기간 서비스 중단, 데이터 손실
   - 장소 정보의 정확성 무보증
9. 분쟁 해결
   - 준거법: 대한민국 법
   - 관할 법원
10. 시행일
```

## 요약: 현재 상태 vs 필요 상태

| 항목 | 현재 | 필요 |
|------|------|------|
| 개인정보 처리방침 | 설정 페이지에 버튼만 존재 (링크 없음) | 문서 작성 + 링크 연결 |
| 이용약관 | 설정 페이지에 버튼만 존재 (링크 없음) | 문서 작성 + 링크 연결 |
| 동의 수집 | 없음 | 로그인 화면에 동의 체크박스 추가 |
| 동의 기록 | DB 컬럼 없음 | `terms_agreed_at`, `privacy_agreed_at` 추가 |
