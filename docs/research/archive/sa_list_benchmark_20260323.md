# SA-LIST Benchmark: List/Collection 기능 비교 분석

> 조사일: 2026-03-23 | 대상: Spotify, Instagram, Letterboxd, YouTube
> 목적: SA-LIST (사우나 장소 큐레이션 리스트) 벤치마킹

## 2. SA-LIST TOP 5 벤치마크 패턴

### Pattern 1: YouTube식 3단계 공개 설정
**출처**: YouTube (public / unlisted / private)
**적용**: SA-LIST도 비공개 → 일부공개(링크 아는 사람) → 공개(피드 노출) 3단계
**이유**: 사우나 리스트는 민감할 수 있음 (내 동네 단골 등). "링크 공유는 하고 싶지만 피드에는 안 올리고 싶다"는 니즈가 분명히 있음.
**SA-LIST 현재**: 공개/비공개 2단계 + 최소 3장소 조건 → unlisted 추가 검토 필요

### Pattern 2: Letterboxd식 아이템별 노트
**출처**: Letterboxd (리스트 내 각 영화에 짧은 코멘트 작성)
**적용**: 리스트 내 각 장소에 메모 (예: "여기 냉탕이 14도", "토요일 오전이 한산")
**이유**: SA-LIST의 핵심 차별점. 단순 북마크가 아닌 "큐레이션"이 되려면 왜 이 장소를 넣었는지 맥락이 필요. 사우너 전문성 표현의 핵심.
**SA-LIST 현재**: 핸드오프에 "메모 CRUD" 언급됨 → 올바른 방향. Letterboxd 수준으로 구현할 것.

### Pattern 3: Spotify식 Save-then-Snackbar 2단계 UX
**출처**: Spotify (♥ → 기본 저장 + 토스트 "추가됨" + 리스트 변경 링크)
**적용**: 하트 1탭 = 기본 리스트 즉시 저장 + 리치 스낵바 "SA리스트에 저장됨 [리스트 선택]" 5초 표시
**이유**: 1탭 저장의 즉시성 + 원하면 2단계로 멀티리스트 선택. 마찰 최소화.
**SA-LIST 현재**: 이미 이 패턴으로 설계됨 (리치 스낵바 5초 + 바텀시트). 올바른 선택 확인.

### Pattern 4: Letterboxd식 에디토리얼 피쳐 ("List of the Week")
**출처**: Letterboxd (스태프가 선정한 주간 베스트 리스트 하이라이트)
**적용**: SA-LIST "발견" 탭에서 운영자 큐레이션 리스트 피쳐링 (예: "이번 주의 SA-리스트", "겨울 노천탕 BEST")
**이유**: ~200개 장소 DB에서 알고리즘 추천은 의미 없음. 소규모 DB에서는 에디토리얼 큐레이션이 훨씬 효과적. Letterboxd도 초기에 이 전략으로 커뮤니티 구축.
**SA-LIST 현재**: "발견" 탭 계획 있음. 알고리즘 대신 에디토리얼 우선 전략 추천.

### Pattern 5: Instagram식 1탭 저장 + 자동 분류 단순성
**출처**: Instagram (북마크 1탭으로 끝, 컬렉션 분류는 선택적)
**적용**: 처음 사용자에게는 "그냥 하트만 누르면 됨". 리스트 생성/분류는 파워유저 기능으로 점진적 공개.
**이유**: 사우나 앱 유저 대부분은 "가고 싶은 곳 저장"만 원함. 리스트 생성은 10% 파워유저. Progressive disclosure 필수.
**SA-LIST 현재**: 기본 리스트 자동 생성 설계가 이에 부합. 좋은 방향.

## 3. SA-LIST에 적용하지 말 것 (Anti-patterns)

| 패턴 | 이유 |
|------|------|
| Spotify AI 추천 | ~200 장소 DB에서 알고리즘 추천은 무의미. 노이즈만 발생 |
| YouTube 자동추가 규칙 | 장소 DB가 작아서 태그 기반 자동추가는 과잉 기능 |
| Instagram 비공개 전용 | SA-LIST의 핵심은 공유/구독. 비공개 기본값은 공유 문화 형성에 역효과 |
| Spotify 협업 실시간 동기화 | MVP에서 실시간 협업은 복잡도 대비 가치 낮음. V2에서 검토 |
| Letterboxd 댓글 | MVP에서 리스트 댓글은 관리 부담. 구독 수로 소셜 프루프 충분 |
