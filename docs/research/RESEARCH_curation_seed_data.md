# 큐레이션 사우나 리스트 — 시드 데이터 전략 리서치 프롬프트

> Google Deep Research용. 아래 4단계를 순서대로 조사하고, 각 단계별로 구체적 결과물을 정리해주세요.

---

## 배경

사우나/찜질방 기록 PWA를 만들고 있습니다. 유저가 장소를 탐색하고 방문 기록을 남기는 앱인데, 현재 DB에 장소 데이터가 거의 없어서 초기 유저 경험이 비어 있습니다. 큐레이션된 사우나 리스트를 먼저 채워서 "오 여기 정보 좋다" "저장해두고 가보고싶다"를 만들어야 합니다.

### 큐레이션 로드맵 (3 Phase)
1. **Phase 1: 한국 🇰🇷** — 핵심 시장. 서울/수도권 + 주요 도시 사우나/찜질방/대중목욕탕
2. **Phase 2: 일본 🇯🇵** — 사우나 문화 선진국. 이키타이 데이터 풍부, 한국인 여행 수요 높음
3. **Phase 3: 해외 Destination 🌍** — 핀란드·독일·발트3국·뉴욕·LA 등 "사우나 여행" 목적지

> 이 리서치에서는 3개 Phase 모두에 대한 소스와 전략을 조사합니다.

### 현재 DB 구조 (참고)
- `places` 테이블: id, country_code(KR), facility_type(gender-bath/male-only/female-only/private-bath/mixed-bath), latitude, longitude, facilities[], is_24h, status
- `place_sources` 테이블: place_id, source(naver/google/manual), external_id, name, address, short_address, phone, url
- 장소 등록 시 Naver/Google Place API로 검색 → 좌표 기반 50m 중복 감지 → 자동 병합
- 장소 유형: 대중목욕탕(gender-bath), 남성전용(male-only), 여성전용(female-only), 개인탕(private-bath), 혼욕(mixed-bath)

## Stage 1: 큐레이션 리스트 기획 & 시드 데이터 소스 조사

### 조사해야 할 것

**A. 한국 사우나/찜질방 큐레이션 소스 전수 조사**

아래 소스들을 각각 조사하여 "리스트 이름 / 장소 수 / 접근 방법 / 데이터 품질"을 정리:

1. **사우나 이키타이 (サウナイキタイ)** — 일본 최대 사우나 포털. 일본 사우나 API나 구조화된 데이터 접근 가능 여부
2. **올해의사우나 (사사사 / 사우나를 사랑하는 사람들의 모임)** — 커뮤니티 기반 추천. 연말 투표 결과, 추천 장소 목록 존재 여부, 접근 경로 - 올해의 사우나 선정을 했었음.
3. **먼데이사우나** — 카카오톡 오픈채팅 커뮤니티, 인스타 계정도 활발히 운영 중. 자주 언급되는 인기 장소, 리뷰가 많은 장소 파악 방법
4. **고독한 사우너 (유튜브/블로그)** — 리뷰한 장소 목록, 평점/추천 체계 유무
5. **네이버 블로그/카페** — "사우나 추천", "찜질방 추천" 검색 시 반복 등장하는 장소들 - 지역별 정리 가능?
6. **구글 맵스** — 한국 사우나/찜질방 카테고리 검색, 평점 4.0+ 장소 추출 가능 여부
7. **카카오맵** — 사우나/찜질방 카테고리, 리뷰 수 상위 장소
8. **트립어드바이저 / 망고플레이트 / 다이닝코드** — 사우나/스파 카테고리 유무, 외국인 대상 추천
9. **관광공사 / Visit Korea** — 외국인 대상 한국 사우나 추천 리스트
10. **인스타그램** — #사우나추천 #찜질방추천 해시태그 인기 장소 패턴
11. **레딧 / 트위터(X)** — r/korea, r/sauna 등에서 한국 사우나 추천 스레드

**A-2. 일본 사우나 소스 조사 (Phase 2)**

1. **サウナイキタイ (사우나 이키타이)** — 일본 최대 사우나 DB. 전체 시설 수, 랭킹 시스템, 지역별 TOP, API/구조화 데이터 접근 가능 여부. 한국 유저가 많이 가는 일본 사우나 (도쿄/오사카/후쿠오카 등) 중심으로 추출 가능한지
2. **サウナランド (사우나랜드)** — 사우나 전문 미디어. 랭킹/추천 리스트 유무
3. **TTNE (サウナシュラン / 사우나슐렝)** — 연간 사우나 랭킹. 역대 수상 시설 리스트, 접근 방법
4. **Google Maps Japan** — "サウナ" 카테고리 평점 4.0+ 장소

## Stage 2: 장소 DB 채우기 전략

### 조사해야 할 것

**A. 데이터 정규화 & 입력 파이프라인**

수집한 장소 데이터를 우리 DB 스키마에 맞추려면:

1. **이름 정규화**: "○○탕", "○○사우나", "○○스파", "○○온천" 등 표기 통일 규칙
2. **주소 → 좌표 변환**: Naver/Google Geocoding API 활용 전략 (쿼터, 비용)
3. **중복 감지**: 같은 장소가 네이버 vs 구글 vs 수동 등록으로 3번 들어올 수 있음. 50m 반경 중복 감지 로직 활용 방안
4. **facility_type 자동 분류**: 이름/설명에서 대중탕/남전/여전/개인탕/혼욕 자동 추론 가능 여부
5. **facilities[] 매핑**: 각 소스의 시설 정보를 우리 27개 시설 항목(HEAT/ICE/PAUSE/BEYOND/AMENITIES)에 매핑하는 규칙

**B. 벌크 임포트 스크립트 설계**

- CSV/JSON → Supabase 벌크 INSERT
- Naver/Google API 자동 검색 → place_sources 연결
- 실패/중복/누락 처리 로그
- 예상 API 호출 횟수 및 비용 추정 (장소 100개 / 300개 / 500개 기준)

**C. 데이터 품질 기준**

최소 입력 필수 필드 vs 있으면 좋은 필드:
- 필수: name, address, facility_type, latitude, longitude
- 권장: phone, is_24h, facilities[], source URL
- 선택: 영업시간, 가격대, 사진

## Stage 4: 큐레이션 리스트 제작 & 운영 전략

### 조사해야 할 것

**A. 초기 큐레이션 콘텐츠 전략**

- 앱 론칭 시 최소 몇 개의 리스트 / 몇 개의 장소가 있어야 "콘텐츠가 있다"고 느끼는지?
- 에디터(우리가 직접) vs 커뮤니티(유저) 큐레이션 비율
- 리스트 업데이트 주기 및 운영 리소스 추정

**B. SEO & 공유 최적화**

- 큐레이션 리스트 페이지가 검색엔진에 노출되려면? (SSR, OG 태그, 구조화 데이터)
- 인스타/카톡 공유 시 프리뷰 최적화
- "서울 사우나 추천" 같은 키워드로 유입될 수 있는 전략

**C. 법적 고려사항**

- 타 플랫폼(네이버/구글/이키타이) 데이터를 우리 DB에 가져올 때 저작권/이용약관 이슈
- 장소 정보(이름, 주소, 좌표)는 사실 정보라 저작권 대상 아님 — 맞는지 확인
- 리뷰/평점/사진은 원저작자 권리 — 가져오면 안 되는 것 목록
- place_sources에 원본 출처 링크를 남기는 것으로 충분한지

## 기대 결과물

1. **소스별 장소 데이터 확보 가능성 매트릭스** — 3개 Phase별로 (소스 × 데이터항목 × 접근방법 × 품질)
2. **추천 큐레이션 리스트 카테고리** — 한국 10개 + 일본 6개 + 해외 5개 이상, 각 카테고리별 확보 가능 장소 수 추정
3. **데이터 수집 → DB 입력 파이프라인 설계안**
4. **리스트 기능 DB 스키마 제안**
5. **초기 운영 전략** (Phase별 몇 개 리스트 × 몇 개 장소로 시작, 어떻게 확장)
6. **법적 리스크 체크리스트** (한국/일본/글로벌 각각)
7. **Phase별 우선순위 장소 리스트 초안** — 각 Phase에서 "이것만은 반드시" 넣어야 할 대표 장소 10~20개씩
