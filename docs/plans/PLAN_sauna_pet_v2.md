## 2. 기획서 ↔ 코드 충돌·중복 매핑 (가장 중요)

### 2.1 펫 종 이름 = 기존 트라이브 첫 칭호 ⚠️
| 트라이브 | 첫 로그 칭호 (기존) | 펫 종 (스펙) | 일치 여부 |
|---|---|---|---|
| saunner | **사우나돌** | 사우나돌 | ✅ |
| bather | **물두꺼비** | 물두꺼비 | ✅ |
| jimi | **구운달걀** | 맥반석란 | ⚠️ 이름 다름 |

함의:
- 트라이브 선택 → 첫 로그 → 그 종이 자동 펫이 되는 흐름이 자연스럽게 가능.
- 스펙은 "온보딩 직후 3종 중 1종 선택"으로 설계. 트라이브 매핑 vs 자유 선택 — **결정 필요**.
- jimi의 칭호 명을 "맥반석란"으로 통일할지 / 칭호와 펫명을 분리할지 결정 필요.

### 2.2 "첫 사-피엔스" 베타 보상 — 칭호와 펫 보상 중복
- 기존: `IS_BETA=true`일 때 `beta_signup` 마일스톤 → 칭호 "첫 사-피엔스" 부여.
- 스펙: 같은 이름의 영구 배지 + 베타 코스튬 + 베타 배경.
- **결정 필요**: 동일 식별자(`has_first_sapiens_badge`) 하나로 통합 vs 칭호와 펫 배지 별개 트랙 운영.

### 2.3 XP 체계 — 스펙 vs 기존 완전 불일치
| 항목 | 기존 (`XP_VALUES`) | 스펙 §6.2 |
|---|---|---|
| 사우나 1회 base | short_log 20 + deep_log 30 | 100 |
| 사이클 보너스 | 없음 | +30/cycle |
| totonou 등급 보너스 | 없음 | 1.0~1.5× multiplier |
| 신규 사우나 | 없음 (place_created 50과 별개) | +50 |
| Weekly cap | 없음 | 800 |
| Streak 4주 | 없음 | +500 |
| 친구 초대 | 없음 | +50 (양쪽) |

함의:
- 그대로 도입하면 누적 XP·레벨이 한 자릿수 → 두 자릿수 점프 → **기존 칭호 마일스톤 일제 폭발**.
- 옵션 A: 글로벌 XP 재설계 + 마이그레이션 (큰 변경)
- 옵션 B: 펫 전용 별도 XP(`pet_xp`) 트랙 신설 — 사용자 레벨과 분리
- **결정 필요**: A vs B (영향 범위 큼)

### 2.4 4주 streak — 신규 메커닉
- 기존: streak 시스템 없음.
- 스펙: M08 "꾸준한 사우나러" 영구 칭호 + 500 XP.
- 함의: streak 평가 엔진 신규 구축. `weekly_streaks` 테이블 + 주별 평가 cron 또는 lazy.

## 4. 보완 리서치 필요 항목

스펙대로 진행하려면 **사용자(스텔라) 또는 디자인/콘텐츠 결정이 필요한** 리서치.

### 4.1 디자인·아트
- **R1**. 아트 스타일 5개 후보 시안 (스펙 §2.3) — 누가 제작? 외주? AI 시안?
- **R2**. 15 캐릭터 SVG + 19 코스튬 SVG + 4 Lottie. 솔로 디자인 가능 시간 산정 → 외주 견적 확보 여부.
- **R3**. L3 9 variant 시각 톤 명확성 검증 (a/b/c 시각 식별 가능?). variant 미니 시안 1세트 사전 검토.

### 4.2 콘텐츠
- **R4**. L3 퀴즈 3문항 — 한국 사우너 페르소나 정합성. 카톡 시드 데이터에서 "사우나 가는 동기" 패턴 추출해 a/b/c 분포 확인.
- **R5**. 코스튬 19개 한국형 모티프 확정 — 면수건 양머리·식혜·한복·자작나무 등. 스펙은 "베이스" 리스트만 있음.

### 4.3 기술 인프라
- **R6**. 추천인 코드 unique 보장 알고리즘 — 8자 영문/숫자 충돌 확률, 재시도 로직. 50명 cap이면 큰 문제 아님.
- **R7**. **인앱 알림 인프라** — 현재 토스트만 존재. 친구 로윌리·L3 진화 준비·미션 완료 알림을 어떻게 표시할지 (벨 아이콘 + 알림 페이지 신규 제작 필요).
- **R8**. Hydration decay cron 또는 lazy 계산 — Supabase scheduled function vs read-time 계산. read-time이 더 가볍지만 친구 목록 정렬 정확도 영향.
- **R9**. 베타 한정 코스튬·배경 cutoff 메커니즘 — `IS_BETA=true` 플래그 종료 시점 결정 필요.

### 4.4 제품 가설 검증
- **R10**. 스펙 §1.3 KPI 목표(L2 도달 40%, L3 진화 15%, 친구 30%) — 현재 알파 데이터로 sanity check 가능한지.
- **R11**. 사사사 협업 시 **시즌 아이템 유료 전환**(SAPI_444_ROADMAP §3-2). 스펙 §1.4 "V1 비목표: 유료 구독/IAP"와 충돌. V2 일정과 동기화 필요.
