# 어드민 페이지 — 구현 스펙 (개발자 친구용)

## 이게 뭐야
운영자(나)가 **장소 데이터**와 **추천 콘텐츠**를 관리하는 내부 페이지예요.
지금은 이 작업이 전부 **코드 스크립트로만** 가능해서, **화면(UI)으로** 만들어야 해요. 경로 `/admin`, 운영자만 접근.

## 먼저 알아둘 배경 3가지 (이거 모르면 헷갈려요)
1. **장소의 이름·주소는 "장소" 자체에 없어요.** 네이버/구글에서 가져온 **원본 소스**에 따로 저장돼 있어요(한 장소에 소스가 여러 개일 수 있음). → 이름·주소 고치기 = 그 소스 데이터를 편집하는 것.
2. **큐레이션** = 운영자가 만드는 **테마 추천 리스트**(예: "전국 노천탕 Top 10"). 앱에서 눈에 띄게 노출돼요. "추천으로 표시" = `is_featured`라는 on/off 플래그.
3. **장소는 2종류**예요. ①우리가 **직접 검수한 정식 장소**(상세 정보 풍부) ②**공공데이터로 자동 들어온 기본 장소**(이름·주소·영업상태만, 수천 곳). 어드민에서 기본 장소를 **노출 on/off** 하거나 검수해서 **정식으로 승격**시켜요.

---

## 만들 화면 4개

### 1. 장소 관리 ⭐️ (가장 핵심)
- 장소 **목록 / 검색**
- 장소 **추가(수동 등록)** · **편집** (시설 종류, 24시 여부, 시설 태그, 위치 등)
- **이름·주소 교정** (위 배경 ①의 소스 데이터)
- **병합(merge)** — 같은 사우나가 중복 등록됐을 때 **하나로 합치기**
- **폐업 처리** (폐업 배지 표시)
- **기본 장소(공공데이터) 운영** — 노출 on/off, 정식 장소로 승격, 일괄 정리
- *(기록 온도 등 데이터 교정 화면 — 기록 시스템 정리가 끝난 뒤에 추가)*

### 2. 큐레이션 관리 ⭐️
- 운영자가 **직접 추천 리스트 생성·편집** (장소 담기, 커버 색·이모지, 제목·설명·태그)
- 유저가 만든 리스트를 **"추천"으로 올리기/내리기** (`is_featured` 토글)
- *(기존 앱의 리스트 생성/편집 로직이 이미 있으니 재사용 가능)*

### 3. 대시보드 (가볍게)
- 핵심 숫자: **유저 수 · 기록 수 · 장소 수 · 트라이브 분포**
- **PostHog**(프로덕트 애널리틱스) · **Sentry**(에러 모니터링) 링크 연결
- **데이터 품질 체크**: 좌표 빠진 곳 · 중복 의심 · 정보가 빈약한 장소 목록

### 4. 인박스 (최소)
- 유저가 보낸 **신고·피드백을 모아 보는 큐** (예: "장소 정보 틀려요 / 폐업했어요", 일반 피드백)
- **읽음 · 처리중 · 완료** 상태 관리
- 받는 쪽 = 앱에 **가벼운 제출 폼**(장소 상세의 "신고" 버튼 + 피드백 폼)도 같이 필요
- ※ **답변은 이메일로** — 앱 내 양방향 메시징은 안 만들어요(복잡도↑)

## 기술 메모 (중요)
- 경로 `/admin/*` — **미들웨어로 접근 차단 + 서버에서 한 번 더 권한 검증**.
- 운영자 식별 = **하드코딩된 ID**(지금은 1인 운영이라 이걸로 충분).
- **쓰기 작업은 권한 높은 키(service_role)가 필요** → 반드시 **서버 쪽(route handler)에서 처리**. 브라우저(클라이언트)에 service key 절대 노출 금지.
- 기존 등록 스크립트(`scripts/katalk-*`)에 **검증·중복방지 로직**이 있으니, 그걸 서버 API로 옮겨 재사용하면 빨라요.
