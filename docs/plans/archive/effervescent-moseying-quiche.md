# 카톡 데이터 분석 → 시드 데이터 + 시설 태그 반영 플랜

## 작업 범위
1. **분석 문서 저장** — 시설별 후기·태그 정보를 별도 문서로 정리 (스텔라 검토용)
2. **seed-data.json 업데이트** — 신규 시설 20곳+ 추가

---

## Step 1: 분석 문서 생성 (`docs/research/katalk-facility-analysis.md`)

커뮤니티 후기에서 추출한 시설별 태그 정보를 정리. 스텔라가 카톡 원본 확인 후 반영할 수 있도록 구성.

### 문서 구조:
```
## 시설별 태그 후보 (검토 필요)
| 시설명 | 추출 태그 | 근거 요약 | 반영? |
각 시설에 대해:
- 매핑 가능한 PLACE_SPECS 태그 ID 나열
- 카톡 원문 근거 요약 (온도, 시설 특징 등)
- 체크박스로 반영 여부 표시
```

### 기존 PLACE_SPECS 태그 매핑:
| 커뮤니티 언급 | 기존 태그 ID | 비고 |
|--------------|-------------|------|
| 냉탕 15-23도 | `cold-bath` | 온도 수치는 태그에 미포함 |
| 급냉탕/얼음탕 | `ice-bath` | |
| 외기욕 | `outdoor-rest` | |

| 건식사우나 | `dry-sauna` | |
| 습식사우나 | `wet-sauna` | |
| 불한증막 | `bulgama` | |
| 아우프구스 | `aufguss` | |
| 셀프 로울루 | `self-loyly` | |
| 세신 | `scrub` | |
| 마사지 | `massage` | |

→ 새 태그 추가 불필요. 기존 23개 태그로 충분히 커버.
