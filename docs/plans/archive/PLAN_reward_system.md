# 기여 보상 체계 설계 (Reward System)

> 상태: 초안 | 작성: 2026-03-01

## 목적

앱이 장려하는 행동을 정의하고, 각 행동에 맞는 보상(뱃지/칭호)을 설계하여 자연스러운 동기 부여 루프를 만든다.

## 장려 행동 분류

### 1. 기록 (Recording)
앱의 핵심 가치 — 꾸준한 기록 습관 형성

| 행동 | 뱃지 아이디어 | 조건 예시 |
|------|-------------|----------|
| 첫 기록 | 첫 발자국 | 로그 1건 작성 |
| 꾸준한 기록 | 루틴러 | 4주 연속 주 1회 이상 기록 |
| 다양한 기록 | 올라운더 | 목욕/사우나/찜질 3타입 모두 기록 |
| 연속 기록 | 스트릭 마스터 | 7일/30일/100일 연속 기록 |

### 2. 장소 기여 (Place Contribution)
커뮤니티 데이터베이스를 풍부하게 만드는 행동

| 행동 | 뱃지 아이디어 | 조건 예시 |
|------|-------------|----------|
| 장소 등록 | 개척자 → 탐험가 → 지도제작자 | 1/10/30곳 등록 (첫 등록만으로 개척자 획득) |
| 지역 다양성 | 전국구 | 3개 이상 시/도에 장소 등록 |
| 정보 보강 | 디테일러 | 장소에 사진/팁/가격 정보 추가 |

### 3. 리스트 & 큐레이션 (Curation)
Spotify Playlist 모델 — 좋은 리스트를 만들고 공유

| 행동 | 뱃지 아이디어 | 조건 예시 |
|------|-------------|----------|
| 리스트 생성 | 큐레이터 | 리스트 3개 이상 생성 |
| 리스트 공개 | 오픈 큐레이터 | 공개 리스트 1개 이상 |
| 구독자 확보 | 인플루언서 | 내 리스트 구독자 10/50/100명 |

### 4. 커뮤니티 (Social)
다른 유저와의 연결, 앱 생태계 활성화

| 행동 | 뱃지 아이디어 | 조건 예시 |
|------|-------------|----------|
| 리스트 구독 | 팔로워 | 다른 유저 리스트 3개 구독 |
| 스토리 공유 | 스토리텔러 | 인스타 스토리 공유 5회 |
| 초기 유저 | 얼리어답터 | 베타 기간 가입 유저 |

## 보상 형태 (가벼운 버전 → 확장)

### Phase 1 — 뱃지 + 토스트 (MVP)
- 프로필에 획득한 뱃지 아이콘 표시
- 획득 순간 토스트 알림 ("🎉 탐험가 뱃지 획득!")
- DB: `user_badges` 테이블 (user_id, badge_id, earned_at)

### Phase 2 — 프로필 강화
- 대표 뱃지 선택 (프로필 옆에 1개 표시)
- 뱃지 컬렉션 페이지
- 다음 뱃지까지 진행률 표시

### Phase 3 — 실질적 보상 (먼 미래)
- 뱃지 보유자 전용 기능 (특별 스토리 템플릿 등)
- 커머스 연동 시 할인/혜택

## 설계 원칙

1. **과도한 게이미피케이션 지양** — 뱃지가 목적이 되면 안 됨, 자연스러운 사용 결과로 획득
2. **달성 가능한 첫 뱃지** — 첫 기록/첫 장소 등록으로 빠른 성취감
3. **랭킹 없음** — 경쟁이 아닌 개인 달성 중심
4. **점진적 확장** — Phase 1부터 시작, 유저 반응 보고 확장

## 아이디어 메모

- 뱃지 시스템 런칭 시 공지/안내 필요 여부 검토 — 인앱 배너, 푸시, 또는 첫 등록 유도 CTA에 "개척자 뱃지 받기" 같은 넛지로 자연스럽게 알릴 수도 있음. 별도 공지 vs 맥락 내 노출 중 어느 쪽이 효과적일지 베타 때 테스트.
- 장소 추가 화면에 진행률 카운터 표시 — "장소 X개 더 추가 시 '개척자' 업적 획득!" 형태. 행동 직전 맥락에서 보상을 미리 보여주는 넛지. 달성 후에는 다음 뱃지(탐험가) 카운터로 자동 전환.

## 구현 우선순위

Phase 1 (기록 + 장소 기여 뱃지)이 P2 백로그에 등록됨.
리스트/큐레이션/커뮤니티 뱃지는 해당 기능 구현 후 추가.
