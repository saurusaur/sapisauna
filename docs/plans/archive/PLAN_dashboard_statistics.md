# 나의 기록 히스토리 (Stats/Dashboard) UI/UX 통합 설계안

## 1. 개요 및 화면 위치 (UI Integration)
별도의 '통계' 탭을 추가하지 않고, 기존 `src/app/history/page.tsx`의 **`[ 캘린더 뷰 ]`를 월간 대시보드로 승격**시킵니다. `[ 리스트 뷰 ]`는 기존의 검색/탐색 기능을 유지하되 가벼운 요약만 덧붙입니다.
핵심 방향은 `가볍게 기록하고, 예쁘게 회고하며, 트라이브별로 깊어진다`입니다.

### 📍 UI 흐름 (User Flow)
1. **진입:** 하단 네비게이션 '기록(History)' 탭 클릭
2. **리스트 뷰 (기본):** 장소명 검색 및 월별 로그 리스트 확인. (월 헤더에 총 방문 횟수만 소소하게 추가)
3. **캘린더 뷰 (통계 통합):**
   *   상단 필터 칩 (전체 / 사우너 / 목욕파 / 찜질파)
   *   **[New] 월간 통계 요약 카드:** 현재 보고 있는 달(Month)과 선택된 필터(Tribe)에 맞는 핵심 통계 노출
   *   **[Update] 월간 회고형 캘린더:** 방문일을 점과 딥로그 링으로 단순하게 표시하는 캘린더 UI
   *   선택한 날짜의 상세 로그 리스트

## 2. 구체적 기능 명세 (다른 에이전트를 위한 구현 가이드)

### Feature 1: Monthly Hero Stats (월간 요약 카드)
*   **위치:** 캘린더 뷰 진입 시, 상단 필터 칩과 달력 컴포넌트 사이.
*   **목적:** 유저가 달력을 넘길 때마다 해당 월의 기록을 직관적인 숫자와 한 줄 요약으로 확인합니다.
*   **구조:** 공통 코어 지표는 가볍고 직관적으로, 트라이브별 지표는 더 nerdy하게 분리합니다.
*   **동적 렌더링 (필터 연동):**
    *   **'전체' 필터:** "3월은 총 **N번** 방문했고, **N곳**을 다녀왔어요."
    *   **'사우너' 필터:** "3월 나의 평균 루틴은 **사우나 N분 / 냉탕 N초~N분 / 휴식 N분 / N회 반복**"
    *   **'목욕파' 필터:** "3월 나의 평균 루틴은 **온탕 N분 / 냉탕 N분 / 쉬는 시간 N분 / N회 반복**"
    *   **'찜질파' 필터:** "3월 나의 평균 루틴은 **찜질 N분 / 쉬는 시간 N분 / N회 반복**"
*   **보조 지표 예시:**
    *   **전체:** 방문한 날짜 수, 방문한 서로 다른 장소 수
    *   **사우너파:** 평균 토토노이, 평균 온도차
    *   **목욕파:** 평균 수질 만족도, 자주 기록한 물 온도대, 세신 경험/만족도
    *   **찜질파:** 평균 휴식 점수, 평균 발한 점수, 매점 이용 여부/만족도
*   **구현 주의점:** 
    *   현재 화면에 렌더링된 달력의 '연/월(Year/Month)' 상태값을 상위 컴포넌트에서 추적할 수 있어야 하므로, 기존 `HomeCalendar` 내부의 월 상태를 부모로 끌어올리거나 `onMonthChange` 콜백이 필요합니다.
    *   트라이브별 루틴 통계는 총 체류 시간이 아니라 `대표 패턴`을 보여주는 방식으로 해석해야 합니다.
    *   `사우너파`는 `사우나/냉탕/휴식/반복`, `목욕파`는 `온탕/냉탕/쉬는 시간/반복`, `찜질파`는 `찜질/쉬는 시간/반복`으로 문구를 다르게 가져가는 것이 자연스럽습니다.
    *   목욕파의 냉탕 루틴은 선택 입력이므로 데이터 커버리지가 충분할 때만 노출하는 것이 안전합니다.
    *   평균형 지표는 최소 표본 수 기준을 두고, 기준 미만이면 "기록이 더 쌓이면 보여드릴게요" 같은 low-confidence state로 처리합니다.
    *   데이터가 없는 달이나 필터일 경우 "아직 기록이 없어요" 등의 empty state 처리가 필요합니다.

### Feature 2: Calendar as a Heatmap (잔디 심기 달력)
*   **위치:** 기존 `HomeCalendar` 컴포넌트 UI 개선.
*   **목적 및 심리학적 UX:**
    *   **시각적 완결성:** 월간 기록이 차곡차곡 쌓이는 감각을 줍니다.
    *   **감정적 보상:** "이번 달 내가 나를 이만큼 돌봤다"는 회고의 만족감을 만듭니다.
    *   **공유 최적화:** 꽉 찬 월간 캘린더는 그 자체로 예쁜 리캡 이미지가 됩니다.
*   **히트맵/캘린더 표시 원칙:**
    *   날짜 셀은 방문 여부를 가장 단순하게 보여주는 것을 우선합니다.
    *   방문한 날에는 해당 트라이브 컬러의 점 1개만 표시합니다.
    *   딥로그가 작성된 경우에는 점 바깥에 얇은 링(외곽선)을 추가해 더 깊게 기록된 날임을 표현합니다.
    *   같은 날 여러 곳을 방문했더라도 월간 셀에서는 추가 점이나 별도 표시를 늘리지 않고, 동일한 시각 규칙을 유지합니다.
    *   복수 방문 여부는 선택한 날짜의 상세 로그 리스트에서 확인하도록 합니다.
*   **디자인 원칙:** 월간 셀은 정보량 극대화보다도 `예쁘고 직관적인 월간 회고` 경험을 유지하는 것이 우선입니다.

### Feature 3: List View Subtle Update (리스트 뷰 디테일)
*   **위치:** 리스트 뷰의 월별 그룹 헤더.
*   **핵심 기능:** 기존 `2026년 3월` 텍스트 옆에 작게 `(총 N회)` 텍스트를 추가하여, 검색/스크롤 중에도 가볍게 누적 수치를 인지할 수 있게 합니다.
*   **추가 시사점:** 리스트 뷰는 정보 탐색용, 캘린더 뷰는 회고용이라는 역할 분리를 더 명확히 합니다.
