## [D3] 달력: HomeCalendar 재사용 분석

### 현재 상태 비교

| 항목 | HomeCalendar | History 달력 |
|------|-------------|-------------|
| 뷰 모드 | 주간(기본) ↔ 월간 토글 | 월간 only |
| 글래스 카드 | ✅ `glass-card` | ❌ `bg-white` |
| 오늘 표시 | 레드 텍스트 (`var(--color-primary)`) | `bg-stone-700` 원형 |
| 선택 날짜 | `bg-[var(--color-primary-light)]` | `bg-stone-100` |
| 타입별 dot | ✅ TRIBE_COLORS | ✅ TRIBE_COLORS |
| 월 네비 | `< 월 >` + "오늘" 버튼 | `< 월 >` (오늘 없음) |
| 스와이프 | ✅ 좌우 스와이프 | ❌ 없음 |
| 이전/다음달 날짜 | ✅ 흐리게 표시 | ❌ 빈칸 |
| 월별 stat | ❌ 없음 | ✅ "3월 · 8회 방문 · 🔥4 🛁3" |
| expand/collapse | ✅ chevron 토글 | ❌ 없음 |

### 재사용 방법

HomeCalendar에 props 추가:

```
defaultExpanded?: boolean   // true면 월간 뷰로 시작 (history용)
hideToggle?: boolean        // true면 expand/collapse chevron 숨김
```

History에서:
```tsx
<HomeCalendar
  logs={typeFilteredLogs}
  selectedDate={selectedDate}
  onSelectDate={setSelectedDate}
  defaultExpanded={true}
  hideToggle={true}         // 히스토리에선 항상 월간
/>
```

→ History의 자체 달력 코드(getCalendarDays, 그리드, dot 등) **전부 삭제**
→ 월별 stat은 HomeCalendar 바깥에 별도 렌더 (기존 위치 유지)

- 달력 로직 중복 제거 (DRY)
- 홈과 동일한 글래스 + 레드 오늘 + 스와이프 UX 자동 적용

## [D6] 메트릭 숫자 강조 — 제안 + 와이어프레임

### 현재 앱 내 메트릭 표시 패턴 분석

| 페이지 | 메트릭 종류 | 현재 스타일 |
|--------|-----------|------------|
| **탐색/장소상세** `explore/[id]` | "또 갈래요 4.2" | `font-bold` + `var(--color-accent)` 레드 |
| **탐색/장소상세** `explore/[id]` | 기록 리스트 (RecordCard) | ScoreBadge만 레드, 나머지 stone |
| **히스토리 리스트** `history` | RecordCard | ScoreBadge 레드, 상세텍스트 stone-400 |
| **기록 상세** `history/[id]` | 온도 (95°C, 15°C) | `font-medium text-stone-700` |
| **기록 상세** `history/[id]` | 토토노이/재방문 점수 | `var(--color-accent)` 레드 |
| **Lovable 퀵로그** | 85°, 15° | 큰 레드 숫자 (입력 폼) |
| **Lovable place details 2** | "95°C · 45분" | 일반 텍스트, 별점만 레드 |

### 패턴 결론

**점수류**(재방문, 토토노이) = 레드 ← 이미 적용됨
**메트릭류**(온도, 시간) = 회색 ← Lovable 기록 리스트와도 일치

### 제안: 3단계 시각 위계

```
┌─────────────────────────────────┐
│   ───── 🔥 사우너파 ─────       │
│                                 │
│  사우나 온도          95°C  ←── (A)
│  냉탕 온도            15°C  ←── (A)
│  세트 수              3세트 ←── (B)
│  토토노이 강도        좋음  ←── (C)
│  ─────────────────────────      │
│                        최고 ←── (C)
└─────────────────────────────────┘

(A) 핵심 메트릭 (온도)
    현재: font-medium text-stone-700
    제안: font-bold text-stone-700 (볼드만 강화, 컬러 유지)
    → 이유: 온도는 "정보"이지 "평가"가 아님. 레드는 평가 전용.

(B) 보조 메트릭 (세트 수, 시간)
    현재: font-medium text-stone-700
    제안: 유지 (변경 없음)

(C) 평가 메트릭 (토토노이, 재방문)
    현재: var(--color-accent) 레드 ← 이미 적용됨
    제안: 유지
```

### 와이어프레임: 변경 전 → 후

```
[ 변경 전 — history/[id] Quick Log 카드 ]

┌────────────────────────────────┐
│  사우나 온도         95°C      │  ← font-medium (500)
│  냉탕 온도           15°C      │  ← font-medium (500)
│  세트 수             3세트     │  ← font-medium (500)
│  토토노이 강도       좋음      │  ← 레드 (accent)
│  ──────────────────────────    │
│                      최고      │  ← 레드 (accent)
└────────────────────────────────┘

[ 변경 후 ]

┌────────────────────────────────┐
│  사우나 온도         95°C      │  ← font-bold (700) ★ 볼드업
│  냉탕 온도           15°C      │  ← font-bold (700) ★ 볼드업
│  세트 수             3세트     │  ← font-medium (500) 유지
│  토토노이 강도       좋음      │  ← 레드 (accent) 유지
│  ──────────────────────────    │
│                      최고      │  ← 레드 (accent) 유지
└────────────────────────────────┘
```

**요약: 온도 값만 bold로 한 단계 올림. 컬러 변경 없음.**
- 레드 = 평가(점수) 전용이라는 규칙 유지
- Lovable 기록 리스트 패턴과 일관
- 시각 위계: 평가(레드) > 핵심 메트릭(볼드) > 보조 메트릭(미디엄)
