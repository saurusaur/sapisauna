# 📂 Project: Sauna App Gamification Engine Design

## 1. 시스템 개요 (Context)

본 시스템은 1인 개발 환경에서 **최소 노력으로 최대 리텐션**을 확보하기 위한 게이미피케이션 엔진입니다.

* **핵심 전략**: 초반 '촘촘한 레벨업'으로 빠른 성취감 부여, 후반 '숙련도(Mastery)' 기반 희소성 강화.
* **UI 제약**: 별도의 프로필 페이지나 사진 업로드 없이 **홈 화면 상단** 및 **기록 후 스토리 카드**에 칭호를 노출함.

## 2. 데이터베이스 구조 (Scalable Schema)

관리가 용이한 단순 확장형 구조를 채택합니다.

* **`users` table**:
* `xp` (bigint): 누적 경험치
* `level` (int): 현재 레벨
* `active_title` (string): 현재 선택하여 스토리 카드/홈에 노출 중인 칭호


* **`user_titles` table**: (유저가 획득한 칭호 인벤토리)
* `user_id` (foreign key)
* `title_name` (string): 획득한 칭호 명칭
* `type` (fixed/random): 확정 타이틀인지 랜덤 조합물인지 구분



## 3. 경험치(XP) 및 레벨링 엔진

유저의 노력과 앱 기여도에 따른 차등 지급 모델을 적용합니다.

### 행위별 경험치 가중치

* **웰컴 보너스**: 50 XP (최초 가입 시 지급)
* **숏 로그**: 20 XP (가벼운 습관 형성)
* **딥 로그**: 80 XP (양질의 데이터 축적)
* **장소 추가**: 120 XP (지도 정보 확장 기여)

### 3단계 조각 함수(Piecewise Function) 레벨업 로직

성장 구간에 따라 다른 수학적 공식을 적용하여 지루함을 방지합니다.

1. **[구간 1] Lv.1 ~ 10 (밀집 단계)**:
* $Next\_Level\_XP = 40$ (고정)
* 숏 로그 2회면 레벨업 가능. 매 레벨업마다 **랜덤 칭호** 지급.


2. **[구간 2] Lv.11 ~ 30 (리듬 단계)**:
* $Next\_Level\_XP = Fibonacci(Level - 9) \times 50$
* 성장이 점진적으로 느려지며, 5단위(Lv.15, 25)에서 **확정 타이틀** 지급.


3. **[구간 3] Lv.31 이상 (권위 단계)**:
* $Next\_Level\_XP = 2000 + (Level - 30)^2 \times 10$ (지수적 증가)
* 마스터 등급의 희소성 유지.



## 4. 칭호 생성 및 보상 시스템

### 랜덤 칭호 (단어 풀 조합)

코드 내 상수(JSON/Array)로 관리하여 구현 공수를 최소화합니다.

* **Prefix(형용사)**: [뜨거운, 은은한, 전설의, 촉촉한, 몽환적인, 끈기 있는, 투명한, 고요한...]
* **Suffix(명사)**: [사우나러, 뢰일리 마스터, 증기술사, 입욕객, 냉탕고수, 찜질 요정, 수질 감별사...]

### 확정 타이틀 (마일스톤)

* **Lv.15**: 숙련된 사우나러
* **Lv.25**: 베테랑 입욕객
* **특수 조건**: 구독자 100명 달성 시 '사플루언서' 즉시 해금

## 5. UI/UX 연동 로직

* **Active 칭호**: 유저가 획득 목록 중 하나를 선택하면 `active_title` 필드 업데이트.
* **홈 화면**: 유저 이름 위/옆에 선택한 `active_title` 텍스트 노출.
* **스토리 카드**: 기록 완료 후 생성되는 이미지 형태의 카드 하단에 유저 닉네임과 함께 칭호를 렌더링.
