1헤더 — 홈과 동일 구현

홈 코드 그대로: 평면 레드 div(var(--color-primary)) + 하단 곡선 SVG path M0,0 H393 V12 C300,30 110,30 0,16 Z (home/page.tsx 61–66). 곡선 부분을 공용 컴포넌트(예: RedCurveHeader)로 추출해 홈·사리스트가 공유. 높이만 286→232px (프로필 카드가 없어서). 타이포도 홈 히어로 동일: Oswald italic 40px.

2사피픽 캐러셀

레드→베이지 경계에 카드가 걸치는 홈 카드슬롯 문법. №1만 -3° 틸트.

3태그 인터랙션 (클릭해보세요)

타일 탭 → 나머지 3개 타일 숨김, 선택 타일이 풀폭 컴팩트 바(태그명+✕)로 축소되고 아래에 카드 블록 펼침.
바 또는 ✕ 탭 → 결과 닫히고 4타일 복귀.
공간을 태그 그리드 1줄 높이로 절약하면서 "지금 뭘 보고 있는지"가 바에 항상 표시됨.

4MY SHELF 위치 — 제안

v1(베타): 사피픽 바로 아래(현재 안). 지금은 태그·리스트 풀이 작아서 BROWSE가 짧음 → 내 선반이 위에 있어도 발견을 안 가림. 재방문 유저의 첫 목적(내 리스트 열기)이 1스크롤 안에 해결됨.

v2(태그별 리스트가 쌓이면): BROWSE를 위로 올리고 선반은 아래로 — 발견 밀도가 커지는 시점에 전환.

별도 섹션(탭/페이지) 분리는 비추천: 저장 리스트 12개 수준에선 탭 하나가 텅 비어 보이고, 내비 깊이만 늘어남. 선반이 화면 한 줄이라 인라인 비용이 거의 없음. 나중에 저장이 30~50개 넘어가면 그때 "전체보기 →"로 전용 페이지를 여는 게 자연스러움.
