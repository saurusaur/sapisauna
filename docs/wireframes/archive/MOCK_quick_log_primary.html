둘 다 입력했을 때 "어느 쪽이 메인인지" 명시. primary_sauna_kind 컬럼 신설. 한쪽만 입력하면 자동 primary, 둘 다 입력하면 사용자가 명시.

① 토글 내부 ✓ 체크마크

입력된 쪽에 ✓ 표시 — 주 이용은 주황, 보조는 회색. 한쪽만 입력 → 자동 주황 ✓, 메시지 없음. 둘 다 입력 → 메시지 표시: "주 이용 사우나를 선택해주세요". 회색 ✓ 클릭 시 주 이용 전환. 한쪽 × 클리어 시 메시지 사라지고 남은 쪽 자동 주황. 저장은 입력만 있으면 항상 가능.

Primary 결정 로직 (확정)
• DB 컬럼: logs.primary_sauna_kind TEXT CHECK (kind IN ('dry','steam')) NOT NULL (DEFAULT 없음, 앱에서 항상 명시).
• 한쪽만 입력 → primary = 그쪽 자동 (UI: 주황 ✓)
• 둘 다 입력 + 미선택 → primary = null → "주 이용 사우나를 선택해주세요" 메시지 + 저장 비활성
• 둘 다 입력 + 사용자 ✓ 클릭 → primary = 선택한 값 (UI: 그쪽 주황, 다른 쪽 회색)
• Story 카드 / History detail 메인 ΔT = primary_sauna_kind 기반 (steam이면 "STEAM TEMP DELTA")
• Stats 인사이트 카드 토글 디폴트 = 해당 로그의 primary_sauna_kind
