Quick Log — 습식 사우나 추가 UX 비교

사우너 탭의 quick log. 두 안 모두 최소 하나 필수, 건식이 디폴트, 양쪽 입력 가능, 동일한 데이터 모델(logs.sauna_temp + logs.wet_sauna_temp). UI 형태만 다릅니다.

① 세그먼티드 탭 + 양쪽 입력 가능

탭 전환 시 이전 값이 보존됩니다. 둘 다 입력하면 양쪽 탭에 ●이 표시되고, 둘 다 저장됩니다. 한쪽만 채우면 deep log에서 반대편 입력을 권유합니다.
건식 사우나 —°C
슬라이더를 터치하면 입력이 시작됩니다.
냉탕 15°C
Deep Log 진입 시:
state = { sauna_temp: null // 건식 wet_sauna_temp: null // 습식 active_tab: dry }

④ 인라인 미니 토글 + 단일 슬라이더

필드 라벨 옆에 작은 토글(건식/습식). 슬라이더는 한 줄만. 토글 전환 시 이전 값 보존 — 양쪽 다 입력 가능. 입력된 쪽엔 × 버튼이 떠서 그 값만 취소할 수 있습니다. 토글 크기는 ×가 있든 없든 고정.
사우나 —°C
슬라이더를 터치하면 입력이 시작됩니다.
냉탕 15°C
Deep Log 진입 시:
state = { sauna_temp: null // 건식 wet_sauna_temp: null // 습식 active_toggle: dry }
두 안의 트레이드오프
• ① 큰 세그먼티드 탭 — 디스커버러빌리티 높음, 두 종류가 동등하게 인식됨. 다른 quick log 필드들보다 시각적으로 무거움.
• ④ 인라인 미니 토글 — 매우 컴팩트, 다른 필드와 시각 무게 동일. 토글이 작아서 못 볼 수 있음 (학습 필요).
• 데이터 모델·저장 로직·deep log 연동은 두 안 모두 동일.
