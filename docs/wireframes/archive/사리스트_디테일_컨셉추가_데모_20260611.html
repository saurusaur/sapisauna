리스트 구독

버튼 — 홈·로그 코드 기준 정리

구독 pill(C·D폰): 현 사리스트 글래스 pill 유지하되, 파스텔 배경에선 흰 글자가 안 보여 잉크색 글자 + 흰 반투명 배경으로 명도 조정 필요(현 white/20은 진한 커버색 기준). 구독중 상태는 D폰(흰 배경+레드+check).
BottomCTA 대안(E폰 하단): 앱 공통 .btn-primary — 레드 풀폭·radius 12px·글로우. 구독을 주 행동으로 키울 거면 이 패턴이 로그 저장과 일관됨. 단 비구독 상태에서만 노출하고 구독 후엔 사라지는 운용 필요.
공유: 네비 아이콘 단독(22px), 파스텔 위라 white/90 → stone-600.

C — 순례길

순서가 의미 있는 리스트에 최강. 단점: 순서 없는 모음형 리스트엔 과한 연출.

D — 스크랩북

도장·수집 세계관(F4)과 연결 — "가본 곳" 체크가 자연스럽게 들어감. 단점: 틸트·테이프가 많아지면 산만, 시설 7개+면 피로.

E — 시트 오버레이

커버가 가장 임팩트 있고 목록 밀도 최고. 메모가 접혀 있어 '큐레이터의 글' 노출은 가장 약함. 단점: 메모가 핵심 가치인 우리 컨셉과는 결이 덜 맞음.

종합 추천

기본 = A 목차형(에디토리얼 톤, 메모 전면 노출), 코스성 리스트만 C 루트형을 변형으로. D의 '방문 도장'은 A/C 어디든 행 우측 작은 도장 배지로 이식 가능.
