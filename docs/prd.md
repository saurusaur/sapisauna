=========================================
      SAUNA LOG APP PRD (v1.9)
=========================================

1. 핵심 컨셉
- 3-Click Logging: 최소한의 터치로 사우나 기록 완료.
- Flex Share: 기록 데이터를 인스타그램 스토리용 감성 이미지로 자동 변환.

2. 온보딩 및 프로필
- 복수 선택 가능: 목욕파/사우너파/찜질파 중 1개 이상 선택
- 템플릿 자동 설정: 마지막 사용 템플릿 또는 첫 번째 선택 프로필로 기본값 지정
- 실시간 전환: 기록 화면 상단 드롭다운으로 템플릿 즉시 변경 가능

3. 사용자 그룹별 기록 항목 요약

[공통 Quick Log]
- 또 올래요 지수 (슬라이더 1~5): 만족도+재방문 의향 통합 지표 ⭐ 추천 시스템 핵심 데이터

[목욕파 (The Bathers)]
- Quick Log (기본): 수질(슬라이더), 온탕 온도(슬라이더), 또 올래요 지수
- Deep Log (선택): 비용, 세신(만족도/가격), 탕 구성(온도 선택입력), 노천탕 유무, 파우더룸 시설, 주차, 동행자, 방문 목적, 자유 메모

[사우너파 (The Saunners)]
- Quick Log (기본): 사우나 온도(슬라이더), 냉탕 온도(슬라이더), 세트 수(카운터), 토토노이 강도(슬라이더), 또 올래요 지수
- Quick Log에 루틴 단계별 시간 추가 선택 가능
- Deep Log (선택): 비용, 루틴 단계별 시간, 시설 태그(외기욕 등), 냉탕 시설, 파우더룸 시설, 주차, 동행자, 방문 목적, 자유 메모

[찜질파 (The Jijimis)]
- Quick Log (기본): 휴식 퀄리티(슬라이더), 청결도(슬라이더), 또 올래요 지수
- Deep Log (선택): 비용(입력), 먹거리(아이콘), 방 종류(태그), 혼잡도(슬라이더), 편의시설(태그), 동행자, 방문 목적, 자유 메모

4. 그룹별 세부 리스트 (Deep Log) - 모든 항목 선택 입력

(공통)
- 동행자: [혼자, 친구, 가족, 연인]
- 방문 목적: [힐링, 운동후, 숙취해소, 데이트, 작업]

(1) 목욕파 상세
- 비용: 입욕료
- 세신: 만족도(5단계), 가격
- 파우더룸: 드라이기(유/무료), 어메니티 구비, 수건 지급 방식
- 자유 메모: 그날의 물 맛, 분위기 등 자유 입력

(2) 사우너파 상세
- 비용: 입욕료
- 시설 태그: [외기욕, 내기욕, 건식, 습식, 아로마, 로류]
- 냉탕 디테일: [수심깊음, 폭포수, 얼음물]
- 파우더룸: 드라이기(유/무료), 어메니티 구비, 수건 지급 방식
- 자유 메모: 분위기, 루틴 후기 등 자유 입력

(3) 찜질파 상세
- 비용: 입장료, 먹거리 지출
- 먹거리 아이콘: [식혜, 계란, 미역국, 맥주, 라면, 빙수 등]
- 방 종류 태그: [불한증막, 소금방, 아이스방, 황토방, 토굴방]
- 혼잡도: [쾌적 / 적당 / 북적 / 자리없음]
- 편의시설: 안마의자, 만화책, 수면실 상태 등
- 자유 메모: 시설 총평, 힐링 포인트 등 자유 입력

5. 자동 수집 데이터 (사용자 입력 불필요)
- 방문 요일: 타임스탬프에서 자동 추출 → "주말에 가기 좋은 곳" 추천
- 방문 시간대: 타임스탬프에서 자동 추출 → "새벽 사우나" 추천
- 재방문 횟수: 동일 장소 기록 카운트 → 암묵적 선호도 지표
- 계절/날씨: 날씨 API 연동 → "비 오는 날 찜질방" 추천

6. 주요 기능 및 UX
- 위치 기반: GPS로 주변 사우나 자동 검색 및 선택, 없을시 지도 검색으로 추가
- 방문시간 자동화: 기록 입력 시간 베이스로 방문시간 기록
- 비용 자동화: 장소 DB 기반 기본 입욕료 자동 세팅 (장소 추가 가능. 신규장소 추가시 일단 주소 임시등록, 어드민 리뷰 목록에 추가됨)
- 자유 메모: 스토리 생성 시 문구 디자인으로 삽입 가능
- 스토리 템플릿: 9:16 비율, 기록 지표가 포함된 디자인 카드 3~4종 제공

7. 기록 완료 화면
- 기록 저장 후 두 가지 액션 버튼 제공:
  (1) 리스트 보기: 히스토리 화면으로 이동
  (2) 스토리 만들기 (공유): 스토리 템플릿 에디터로 이동

8. 장소 DB 구성
- Naver Maps API (우선): 한국 내 목욕탕/찜질방 데이터 풍부
- Google Places API (보조): 네이버 검색 결과 부족 시 보완
- 검색 로직:
  (1) 네이버 지도에서 우선 검색
  (2) 결과 부족 시 구글로 보완 검색
  (3) 둘 다 없으면 사용자 직접 등록 → 어드민 리뷰 큐
- 중복 제거: 동일 장소 2회 노출 방지 처리
- 주의: 네이버 API는 국내 서비스 전용, 각 API 이용약관 준수 필요

9. 로드맵
- Phase 1 (MVP): 온보딩 + Quick Log + 저장
- Phase 2: 스토리 템플릿 생성 + 공유
- Phase 3: Deep Log + 히스토리
- Phase 4: 장소 DB 고도화 + 어드민 기능
- Phase 5: 추천 시스템 (또 올래요 지수 + 자동 수집 데이터 + 맥락 태그 기반)
