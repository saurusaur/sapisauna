## 1. 관통 컨셉
**사우나에 다녀오면 '무드 젤리'가 하나 쌓인다.** 한 달이 지나면 그게 모여 "이 달의 사우나"가 되고, 각 기록을 탭하면 그날의 추억이 펼쳐진다.
- 도장 "찍기"보다 **모으는 재미**(수집 본능) — 혼자 써도 채워져서 **콜드스타트에 강함**.
- 회고(좋았다/별로였다)·공유까지 자연스럽게 연결.

## 3. 결정된 것 (이 선 안에서 디자인)
- **수집 단위 = 무드 젤리 (v1)** — 트라이브 색 젤리: 🔥사우나파 / 🛁목욕파 / 🥚찜질파. **커스텀 아트 없이** 그라데 원형으로 시작.
  - **확장(나중) = 사우나템** — 디자인 에셋 쌓이면 때수건·식혜·맥반석란 등(=기존 사우나펫 리워드 재활용)으로 젤리를 교체/추가. *데이터는 generic 저장 → 렌더만 교체.*
- **비주얼 = 캔디/젤리/글로우 sub-톤** — 게임화·회고 화면에만. 앱 전체 톤(베이지·레드·글래스·Oswald)은 유지, 글래스모피즘이 다리.
- **F4(도장판) + F5(히스토리)는 한 화면으로 통합 지향** — 이 컬렉션 화면이 곧 회고 화면.

## 4. 열어둔 것 (← 전문가가 전체 그림으로 잡아주세요)
- **메인 메타포**: 🍬유리병? 📅앨범/그리드? 🏅뱃지 갤러리? 혹은 조합(병=월 요약 / 그리드=상세)?
- **전체 IA·동선**: 수집(채워짐) → 회고(탭하면 추억·좋았던/별로였던) → 공유. 기존 KPI·인사이트·리스트뷰 토글을 어디에 둘지.
- **'좋았던/별로였던 한눈에'** 표현 — F3 비교 랭킹과 시각적으로 어떻게 연결할지.

## 5. 제약·연결 (브리프 경계)
- **트라이브 색**(🔥🛁🥚)이 수집물 색 체계의 기본.
- **v1은 커스텀 아트 0**으로 구현 가능해야(빌더 즉시 빌드 조건). 화려한 아트는 v2.
- **UI 5상태**(Empty=첫 사용 가이드 / Loading / Partial / Error / Ideal) 포함.
- 유지: 캘린더↔리스트 토글·KPI(기록한 날·다녀온 곳)·인사이트·최근기록.
- 산출물 = **바로 구현 가능한 와이어**(구현은 우리). 무드보드 수준이 아니라 구조까지.
