① 워터마크 축소

120px → 88px, opacity .3, 헤더 우하단(곡선에 살짝 물림). 타이틀·설명 영역과 안 겹치는 위치. owner는 워터마크 탭 → 이모지·hue 피커(ListFormSheet).

② 구독 + 공유 한 줄

공유 = 구독 pill과 같은 글래스 문법의 32px 원형 pill, 오른쪽 나란히. 방문자 네비에선 share 제거(중복). owner 네비는 현행 유지(공개 pill·share·more_vert) — owner에겐 구독 줄이 없어서.

③ 색상 = 전부 cover_hue 파생 (네, 맞습니다)

유저가 고른 hue 하나에서 4개 톤을 함수로 생성:

헤더 bg
L 88%
액센트
L 60%
틴트 박스
α 0.3
점선
L 78%

헤더 bg는 기존 listBgColor(cover_hue) 그대로, 나머지는 같은 hue에 S/L만 바꾼 파생(예: hsl(hue 40% 55%) 액센트). 구현은 페이지 루트에 CSS 변수 --list-accent 등 4개를 한 번 세팅하면 끝 — 두 폰이 그 증명(같은 컴포넌트, hue 값만 다름). 메모 박스·점선·장소 추가·라벨·SNS 빈 칩 전부 자동으로 따라감.
