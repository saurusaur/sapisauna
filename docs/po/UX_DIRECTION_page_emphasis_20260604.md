# 사-피 UX 방향성 — 페이지별 강조 기능 + 위임 구조 (2026-06-04)

> 목적: 방금 확정한 방향성(구현스케치 `사피_제안기능_구현스케치.html` + BACKLOG F1~F6·어드민·스키마)을 **페이지별로 "뭘 강조할지" 정렬**하고, **선행(우리가 먼저 잠글 것) → 위임(친구들에게 넘길 것)** 구조로 분해. 헤드라인 수준, 디테일은 항목별 별도 플랜.
>
> 3대 축: **페르소나(트라이브) → 발견(추천·큐레이션) → 기록·회고**. 시장 1순위 미충족 니즈 = **사우나 추천(질문 38%)**, 고유가치 = **최고의 사우나 루틴 경험**.

## 1. 페이지별 강조 기능 (UX 재검토 대상)

| 페이지 | 라우트 | 🎯 강조해야 할 핵심 | UX 재검토 포인트 | 관련 |
|---|---|---|---|---|
| 홈 | `/`,`/home` | "지금 뭐하지?" 허브 — 빠른기록 진입 + 발견(트라이브픽/FEATURED) + 도장판 현황 | 발견·기록·회고 중 첫 화면 우선순위 재정의 | F1·F2·F4 |
| 발견(지도) | `/explore` | **사우나 추천** = 트라이브 픽 필터 + 영업중·24시 + 실시간 카운트 | 검색 흐름(센터링·이지역검색·자동완성) 개선 | **F2** |
| 장소 상세 | `/explore/[id]` | 경험 퀄리티 지표(시설·온도·리뷰) + 저장 + 기록 CTA | 시설정보 표시 순서 재배치 | F6·리뷰 |
| 기록 입력 | `/log`,`/log/deep` | **블록 탭 빠른기록(순서=루틴)** 메인, 온도·세트는 상세 토글 | 폼 전면 재설계(선행: 스키마) | **F1** |
| 내 기록 | `/history`,`/history/[id]` | 회고(KPI·인사이트) + 캘린더 재설계 + 비교·랭킹 | '달력' 부분만 재설계, 토글 유지 | **F5**·F3·F4 |
| 사-리스트 | `/sa-list`,`/[id]`,`/my`,`/tribe/[type]` | 테마 큐레이션 발견 + 큐레이터 전환 | 저장→자동 리스트화(후순위) | F6·큐레이션 |
| 스토리 | `/story` | 기록 공유(스토리 카드) → 디스커버리·유입 | 공유 후 재유입 동선 | 공유 |
| 프로필/설정 | `/settings/*` | **사우나 ID 카드**(트라이브·선호·칭호·통계) + 도장판/칭호 | ID 카드 컨셉 설계 | 사우나ID·F4 |
| 온보딩 | `/onboarding` | 페르소나(트라이브) 설정 = 모든 맞춤의 시작 | 트라이브별 기본블록 연결 | F1 |
| **어드민(신규)** | `/admin` | 운영: 등록/교정·큐레이션 관리·신고/폐업 큐·통계 | **구성·기능 스코핑 먼저** | 어드민 |

## 2. 선행 (우리가 먼저 잠근다 — 위임 불가, 기반)
> 이게 안 잠기면 하위 작업이 흔들림. **delegation 전 반드시 선결.**

- **L1. 로그·스키마 구조 재검토** (P0) — 블록/루틴 시퀀스 데이터 모델 결정. F1·F3·F5의 토대.
- **L2. UX 방향 확정** — 본 문서(페이지별 강조 + 정보구조 IA) 합의.
- **L3. F1 블록 기록 인터랙션·데이터 스펙** — 블록 세트·순서·온도분리·추천루틴.
- **L4. 어드민 페이지 구성 스코핑** — 기능 목록·화면·권한 정의.

## 3. 위임 가능 (선행 락 후 친구들에게)
> 각 묶음은 선행 의존성 충족 시 **병렬 진행 가능**. (담당자 배정란은 추후)

| 묶음 | 작업(헤드라인) | 선행 의존 |
