# 스키마 출발점 초안 — 빌더 F4(수집)·F3(랭킹)

> ⚠️ **확정본 아님 = 대화 출발점.** 표 모양은 빌더가 F4/F3를 *어떻게 구현할지* 설계한 뒤, 그 설계에 맞춰 **우리가 확정·적용**한다(마이그레이션 작성·실행은 우리 몫=DB 소유).
> 목적: 빌더가 백지에서 시작 안 하게 + 우리가 생각하는 데이터 방향(특히 generic 저장)을 미리 공유.
> 적용 순서: **빌더 설계 → 이 초안과 대조·수정 → 번호 부여(032~) → 우리가 적용.** (data_tier(031)만 먼저 적용됨)

---

## F4 수집/도장판 — `collection_items` (초안)
"누가 / 언제 / 어떤 방문에서 / 무슨 젤리를 모았나" — 도장판이 채워지는 원천.

```sql
create table collection_items (
  id          uuid primary key default gen_random_uuid(),
  user_id     uuid not null references auth.users(id) on delete cascade,
  log_id      uuid references logs(id)   on delete set null,  -- 어떤 방문이 획득시켰나
  place_id    uuid references places(id) on delete set null,
  item_type   text not null default 'mood-jelly',  -- v1='mood-jelly' → 후일 'goods'
  item_key    text not null,                        -- v1=tribe('saunner'|'bather'|'jimi') → 후일 'ttaesugeon'…
  earned_on   date not null default current_date,   -- 월간 도장판 그룹핑
  created_at  timestamptz not null default now()
);
-- + index(user_id, earned_on), RLS(user_id=auth.uid())
```
**핵심 = generic 저장**: `item_key`만 바꾸면 젤리→아이템 확장(마이그레이션 X).
**빌더와 확인할 것**: 수집 단위(방문당 1개 vs 하루 1개) · 완성/리워드를 별도 표로 둘지 · 시즌 테마를 어디에 기록할지.

## F3 Beli 랭킹 — `place_scores` (+ 비교 이력) (초안)
"누가 / 어떤 사우나 / 몇 점" + (선택) 비교 이력.

```sql
create table place_scores (
  id uuid primary key default gen_random_uuid(),
  user_id  uuid not null references auth.users(id) on delete cascade,
  place_id uuid not null references places(id) on delete cascade,
  score    numeric(4,2),   -- 0~10 개인 점수
  rank     int,            -- 정렬 위치(선택)
  updated_at timestamptz not null default now(),
  unique (user_id, place_id)
);
-- (선택) place_comparisons(user_id, place_a, place_b, winner, created_at) — 재계산·투명성용
-- + RLS(user_id=auth.uid())
```
**빌더와 확인할 것 (핵심)**: 점수 *산출 알고리즘* — 비교만 저장하고 점수는 계산? vs 점수를 갱신 저장? (Beli식 이진삽입/일라이즈 방식 등 빌더 경험에 맡길 영역) → 알고리즘이 정해지면 표가 확정됨.

## 정리: 표는 설계를 따라간다
- **설계 무관 → 지금**: `places.data_tier` (`031`, 적용 완료 가능).
- **설계 의존 → 빌더 설계 후**: `collection_items`(F4) · `place_scores`(F3). 위는 출발점일 뿐.
