# SA-LIST 리디자인 컨셉 (2026-06-10)

레퍼런스: 팟캐스트 앱 모자이크 그리드 (파스텔 카드 + 2톤 타이포 + 추상 도형 + 하단 플로팅 카테고리 내비)

## 레퍼런스에서 가져올 핵심 문법

1. **모자이크 그리드** — 2열 masonry, 카드 높이 가변(콘텐츠 길이/중요도 따라). 현재 Featured 캐러셀의 staggered offset(HOME_STAGGER)을 세로 그리드로 확장한 개념.
2. **2톤 스택 타이포** — 제목 1행: 잉크색 extrabold / 2행: 카드 hue의 진한 톤. 예: "주말 노천탕"(잉크) + "서울 근교 6곳"(톤다운). `cover_hue` 기반이라 기존 데이터로 즉시 가능.
3. **카드별 추상 도형** — 팟캐스트 앱의 원/반원/블롭 자리에 **사우나 모티프**(돔, 수증기 곡선, 탕 단면, 온도계 호)를 hue 파생색으로. SVG 4~6종을 list id 해시로 배정 → 커스텀 에셋 없이 전 리스트 커버 자동 생성.
4. **하단 플로팅 pill 내비** — 이미지의 "Top Shows 993 · Comedy 712" 부분. 카테고리 칩 + 카운트.

기존 디자인과의 접점: 파스텔 카드 bg = `listBgColor(cover_hue)` 그대로, 잉크색 = #2a2222, 베이지 바탕 유지, 섹션 헤더는 home의 `text-[23px] font-extrabold italic font-heading` 유지.

## 컨셉 A — 「사우나 가판대」 (Newsstand Mosaic) ★ 레퍼런스 충실형

**아젠다: 페이지 전체가 '발견'. 잡지 가판대처럼 커버를 훑으며 고른다.**

```
┌─────────────────────────────┐
│ SA-LIST          [＋만들기]  │ ← 컴팩트 헤더
│ [🔍 리스트·태그 검색      ]  │ ← 검색 pill (탭→오버레이)
├─────────────────────────────┤
│ ┌────────┐ ┌──────────┐    │
│ │주말     │ │냉탕       │    │ ← 2열 masonry
│ │노천탕   │ │수련회     │    │   카드 = cover_hue 파스텔
│ │ (돔도형) │ │ (수증기)  │    │   2톤 스택 타이포
│ │@큐레이터│ └──────────┘    │   하단 = 큐레이터·구독수
│ └────────┘ ┌──────────┐    │
│ ┌────────┐ │…         │    │
├─────────────────────────────┤
│  ( FEATURED 23 | #태그 | 저장됨 12 ) │ ← 플로팅 하단 pill
└─────────────────────────────┘
```

- **Featured 피드** = 메인 모자이크 그리드 자체. 어드민 is_featured가 상단 대형 칸(2열 중 1.4배 높이) 차지.
- **검색** = 헤더 pill → 풀스크린 오버레이: 최근 검색 + 인기 해시태그 칩(파스텔 타일로, 태그도 작은 모자이크) + 자동완성.
- **저장됨** = 하단 플로팅 pill의 세그먼트. 탭하면 그리드가 내 저장 리스트로 스왑(페이지 이동 없음).
- 리스크: 하단 pill이 기존 둥근 글로벌 내비와 충돌 가능 → pill을 내비 위 32px 띄움 or 상단 sticky 탭으로 대체.

**난이도 중 / 임팩트 상.** 카드 컴포넌트 1종 + SVG 도형 세트만 만들면 됨.

## 컨셉 B — 「온천 매거진」 (Editorial Issue) — home 연속성형

**아젠다: home의 '레드 돔 + 기울어진 카드' 문법으로 매주 발행되는 잡지 한 호.**

```
[컬러 돔 헤더 — "DISCOVER" 40px italic]
  └ 이주의 사피픽 리스트 1개 = 대형 커버카드 (-3.5deg 틸트, home ID카드 문법)
[해시태그 레일 — 가로 칩, hue 틴트]
[FEATURED 모자이크 — 2열, 컨셉 A 카드 축소판]
[MY SHELF — 저장한 리스트 가로 선반 (책등 메타포: 세로 슬림 카드)]
```

- 검색은 헤더 아이콘 → 오버레이 (A와 동일).
- home과 가장 일관적, "이주의 사피픽"과 직결. 대신 발견 밀도(한 화면 리스트 수)는 A보다 낮음.
- **난이도 중 / 발견 아젠다 중.** 큐레이션 운영(주간 픽 갱신)이 전제.

## 컨셉 C — 「수증기 탐색대」 (Steam Search-first) — 검색 주도형

**아젠다: 해시태그가 곧 진열대. 태그를 먼저 고르고 리스트를 만난다.**

```
[베이지 헤더 — 대형 검색바 hero]
[해시태그 모자이크 — 태그 자체가 파스텔 타일 ("#노천탕 / 42 lists")]
[선택 태그의 리스트 피드 — 모자이크]
[저장됨 — 상단 sticky 탭 (탐색|저장됨)]
```

- 레퍼런스의 카드 문법을 **태그**에 적용. 태그 데이터가 빈약한 초기엔 빈약해 보일 리스크.
- **난이도 하 / 베타 시점엔 시기상조** (태그 풀·리스트 수가 적음).

## 추천

**A 기본 + B의 '이주의 사피픽' 대형 칸을 그리드 최상단에 흡수.**
- 모자이크 첫 칸(대형) = 이주의 사피픽 → home 캐러셀과 스토리 연결 유지
- 검색 오버레이에 C의 해시태그 타일을 축소 수용 → 태그 풀이 자라면 자연 확장
- 저장됨은 하단 pill 충돌 검증 후 결정 (pill vs 상단 탭)
