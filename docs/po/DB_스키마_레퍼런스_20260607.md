## 👤 유저

### `users` — 프로필 (PK=auth.users.id)
| 칸 | 담는 것 |
|---|---|
| `id` | auth 유저 ID |
| `nickname` `gender`('male'\|'female') | 닉네임·성별(탕구분 계산용) |
| `user_types[]` | 선택 트라이브 순서 예: `{saunner,bather,jimi}` |
| `primary_type` | 대표 트라이브(홈/퀵로그 기본값) |
| `xp`(bigint) `level` `active_title` | 리워드 |
| `profile_emoji` `profile_hue`(0~360) | 프로필 아이콘 (※hex는 020에서 폐기, hue 사용) |

### `user_titles` — 칭호
`id · user_id · title · source`('milestone'\|'random'\|'welcome'\|'beta')· `base_title`(마일스톤 원본명)· `granted_at`

### `public_profiles` (뷰) — 비로그인 노출용
`id · nickname · active_title · primary_type · profile_emoji · profile_hue`. **gender 등 비공개 칸 차단.** 리스트/장소에서 owner 표시에 사용.

## 📍 장소

### `places` — 시설 메타 (※이름/주소 없음 → `place_sources`)
| 칸 | 담는 것 |
|---|---|
| `id` `country_code` `city`(Google locality, null가능) | |
| `latitude` `longitude` `coordinate_source`('naver'\|'google'\|'manual') | 대표 좌표 |
| `facilities[]` | 시설 태그 평탄 배열(PLACE_SPECS id) |
| `is_24h` | 24시 여부 |
| `facility_type` | **7종**: small-bath·public-bath·hotel-premium·resort-spa·private-sauna·special·gym-sauna |
| `bath_policy` | gender-bath(기본)·male-only·female-only·mixed |
| `status` | active·closed_permanently·closed_temporarily·under_review |
| `merged` `created_by` | 병합 이력·등록자 |
| `data_tier` 🆕 | **rich**(검수 312)\|**basic**(공공데이터 5천) — 031, 추천/픽/랭킹은 rich만 |

### `place_sources` — 소스별 이름·주소 (place당 N개)
`id · place_id · source`('naver'\|'google'\|'manual')· `external_id` · `name_original` · `address_original` · `latitude/longitude` · `plus_code`. **UNIQUE(source, external_id).** 표시용 name/address는 여기서 조인.

## 📝 기록 (★ 029로 재편 중)

### `logs` — 방문 1건 = 1행 (Quick + 상세 흡수)
**코어**: `id · user_id · place_id · tribe_id · revisit_score`(1~5, "또갈래요")· `record_date`(유저지정 방문 일시, 로컬TZ)· `bath_gender`(자동계산)
**루틴 시간/세트**: `heat_time`(분)· `ice_time`· `rest_time`· `repeat`(1~7세트)
**온도(통일명, 029)**: `dry_sauna_temp`(건식)· `steam_sauna_temp`(습식)· `primary_sauna_kind`('dry'\|'steam')· `hot_bath_temp`(온탕)· `very_hot_bath_temp`(열탕)· `cold_bath_temp`(냉탕)· `ice_bath_temp`(급냉)· `bulgama_temp`(불가마)· `salt_sauna_temp`· `open_air_bath_temp`· `ice_room_temp`
**평가**: `totono_score`(토토노이)· `water_quality`· `rest_quality`· `sweat_quality`· `cleanliness`
**상세(deep_logs서 흡수)**: `crowd · companion · cost · currency · memo`
**세신/마사지(분리, 신규)**: `scrub_score·scrub_cost·scrub_type`('basic'\|'withmassage') / `massage_score·massage_cost`
**매점/식당(신규)**: `snack_score·snack_memo` / `restaurant_score·restaurant_memo`
> 라이브 logs = **총 50칸** (신·구 공존).
>
> ⚠️ **사용 금지 / 폐기 예정 칸** (새 작업은 절대 쓰지 말 것):
> - `sauna_temp·jjim_temp·pause_time` → 030에서 DROP 예정. 대신 `dry_sauna_temp·bulgama_temp·rest_time`.
> - `scrub_types[]·scrub_satisfaction·store_score·store_memo` → **구 029 잔재**(030으로도 안 지워짐, 라이브에 남아있음). 대신 `scrub_type·scrub_score / snack_score·snack_memo` 사용.

### `log_blocks` — 순서·반복·심도의 정본 ★빌더/기록 핵심
한 방문의 활동을 **행 1개=블록 1개**로. 같은 활동 여러 번 OK(seq로 구분).
| 칸 | 담는 것 |
|---|---|
| `log_id` `seq` | 어느 로그 · **순서**(누른 순=루틴) |
| `block_type` | 활동 id (dry-sauna·cold-bath·rest·scrub·snack…) |
| `category` | heat\|ice\|rest\|beyond |
| `temp` `duration_sec`(초통일) `score` `cost` `memo` `variant` | **occurrence마다 독립** → 매 반복 다른 값 가능 |
> 예) `a b a c d` = 5행, a가 seq 1·3. 1번째 a와 3번째 a의 temp/시간 따로 저장.

### `user_routines` — 나만의/추천 루틴
`id · user_id`(NULL=추천 시드)· `tribe_id · name · icon · blocks`(jsonb 배열)· `is_featured`

### ~~`deep_logs`~~ — 폐기 중 (030에서 DROP 예정)
구 상세 테이블. 029가 logs로 흡수 완료 → **신규 작업은 참조 금지**, logs 통일 칸 사용.

## ⚠️ 작업 시 꼭 알 것
1. **이름·주소는 `places`에 없음** → `place_sources`에서 조인(소스별 N개, naver/google).
2. **추천·트라이브픽·랭킹·피드 쿼리**: `data_tier='rich'` + 어드민 시드(23c4…) 제외 + `status='active'`.
3. **온도 단위**: 전부 °C(정수). 블록 `duration_sec`만 초, logs 시간 칸은 분.
4. **순서/반복/심도가 필요하면 `log_blocks`** (logs 온도 칸은 대표값 캐시).
5. **RLS 전부 켜짐** — 유저는 본인 것만. 어드민 쓰기·집계는 service_role(서버 route handler).
6. 컬러는 **hue(int 0~360)**, hex 아님(020/019).

## 🔑 DB 접근 권한 (네가 할 것)
Supabase 대시보드 → **Organization → Members → Invite** (읽기 위주면 권한 낮게). 또는 친구가 직접 안 보고 이 문서 + 우리가 뽑아주는 샘플로 작업해도 됨. 쓰기 작업(어드민 등)은 어차피 **서버 route handler + service_role**라 친구가 service key 직접 만질 일은 없음.
