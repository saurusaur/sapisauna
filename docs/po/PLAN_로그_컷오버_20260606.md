- **D3 수동 탕 선택(bath_gender)** → ✅ **자동 + 헤더 '변경'에서 수정**(입력폼 X). UX = §C-1.
- **D4 매점/식당** → ✅ **점수 + 메모(추천메뉴)**: v6 우측칸 = ₩가격 대신 **메모 입력**(placeholder "추천메뉴 예: 식혜가 시원"). 가격 컬럼 추가 안 함. `snack_score/snack_memo`, `restaurant_score/restaurant_memo`에 저장. food_eaten = 폐기(이미 고아).
- **D5 세신/마사지** → ✅ **`scrub_types[]` 폐기 + `scrub_type`(종류) 신설**:
  - 팔레트 3 도장: **세신**(`block_type=scrub, variant=일반세신`) · **건강세신**(`scrub, variant=건강세신`=마사지 포함) · **마사지**(`massage`, standalone).
  - 저장: `log_blocks.variant`=세신 종류. 캐시 `scrub_score/scrub_cost/scrub_type` + `massage_score/massage_cost`는 블록서 파생. → "마사지만 했는데 scrub_*" misleading 제거.
  - variant 값 = `'basic'`(일반세신) / `'withmassage'`(건강세신=마사지세신). 각 블록 만족+₩가격 1세트(scrub→scrub_cost, massage→massage_cost). **자동태깅: 건강세신 → `scrub`만**(massage 태그 안 함).
  - **029 재적용 완료**: log_blocks 전체 삭제→분리규칙 재생성(중복·이중계산 원천 제거). 둘 다→scrub(withmassage), 세신만→scrub(basic), 마사지만→massage.

### C-1. D3 '변경' = 인라인 확장(컴팩트, 확정)
변경 누르면 헤더 아래 확장 → 날짜·시간·탕을 **한 행 작은 드롭다운**으로 묶고, 장소 재선택은 링크(→검색, 드래프트 보존). 한 줄씩 X, 중복 없이 컴팩트. 탕=자동값 기본 + 드롭다운 override.
