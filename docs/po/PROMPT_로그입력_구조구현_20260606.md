# 프롬프트 — 사우나 로그 입력 화면 "구조/인터랙션" 구현

> 아래 전체를 다른 창(프로토타입/코드 생성 AI, 또는 디자인 툴)에 붙여넣어 사용하세요.
> **비주얼 룩(색·폰트·그림자·간격·아이콘 스타일)은 직접 입힐 예정**이므로,
> 여기서는 **레이아웃 골격 + 동작 로직 + 상태 모델**만 정확히 구현하면 됩니다.

---

## 목적

모바일 PWA "사-첵"(사우나/찜질방 1세션 기록) **로그 입력 화면 한 장**의 구조와 인터랙션을 구현한다.
핵심은 (1) **내 루틴 상세 토글**로 초보/고급 모드를 가르고, (2) 시설 팔레트를 **카테고리별 가로 스크롤 레인**으로 컴팩트하게 펼치는 것.
단일 화면(세로 스크롤), 모바일 폭 ~360px 기준.

## 화면 구성 (위 → 아래)

1. **상단 요약 헤더** (편집 가능)
2. **블록 선택 영역** — "오늘 뭐 했어요?"
   - 트라이브 기본 블록 (항상 노출)
   - `＋ 전체 시설 더보기` → **가로 스크롤 레인** (펼침)
3. **내 루틴 상세 토글** (온/오프) — 선택한 블록의 표현 방식이 바뀜
4. **퀵 평가** — 트라이브 품질 + 또갈래요(재방문)
5. **더 자세히** (접힘)
6. **저장 버튼** (하단 고정 느낌)

---

## 1. 상단 요약 헤더 (편집 가능)

- 한 줄 카드: `📍 시설명` · `날짜 · 시간 · (자동)탕 성별`
- 우측 `변경` 버튼 → 시설/시간을 수정하는 진입점 (실동작은 시설검색/시간피커로 연결, 프로토에선 토스트/모달 stub 가능)
- 시설명/시간은 **요약이자 편집 진입점** — 항상 보이되 거슬리지 않게 한 줄로

## 2. 블록 선택 영역

### 2-1. 트라이브 기본 블록 (항상 노출)
- 유저의 트라이브(사우나파/목욕파/찜질파)에 따라 **기본 5블록**을 가로 한 줄로 항상 노출
- 각 블록 = 아이콘 + 라벨. **탭 = 선택 토글**. 선택되면 **순번 배지**(1,2,3…) 표시
- 순번 = 탭한 순서 (= 동선 순서)

### 2-2. `＋ 전체 시설 더보기` (발견성 핵심)
- **풀폭, 큼직한** 버튼. `전체 시설 더보기` + `19종` 같은 개수 배지 + 펼침 셰브론
- 탭 = 아래 가로 레인 영역 펼침/접힘 (셰브론 회전)

### 2-3. 가로 스크롤 레인 (D 패턴 — 컴팩트 확장의 핵심) ★
- 펼치면 **카테고리 4줄로 "세로 고정"** (창이 세로로 길어지지 않는 게 목적)
- 4 카테고리: **heat(열) / ice(냉) / rest(휴식) / beyond(그외)**
- 각 줄(lane) 구조:
  - **좌측 고정 태그**: 카테고리 아이콘 + 라벨(열/냉/휴식/그외) + 그 줄 항목 개수
  - **우측 가로 스크롤 영역**: 그 카테고리 블록 칩들이 가로로 나열, **옆으로 스크롤**
- 가로 스크롤 영역 필수 사항:
  - `overflow-x:auto`, 스크롤바 숨김, `scroll-snap-type:x proximity`
  - **우측 페이드 그라데이션 + 화살표(chevron_right)** 로 "더 있음" 암시 (화살표 살짝 nudge 애니메이션)
  - **끝까지 스크롤하면 페이드/화살표 사라짐** (scroll 이벤트로 끝 감지)
- 칩 = 아이콘 + 라벨. **탭 = 선택 토글**, 선택 시 칩 안에 **순번 배지**
- **선택 상태 동기화**: 기본 블록(2-1)과 레인 칩이 같은 블록이면 선택/순번이 양쪽에 함께 반영

## 3. 내 루틴 상세 토글 (온/오프) ★ 신규 요구사항

선택한 블록을 **어떻게 보여주고 무엇을 더 입력받는지**를 토글로 가른다.

- 토글 스위치 1개. 라벨 예: **"순서·온도·시간 기록"** 또는 **"내 루틴 상세"**. 기본값 **OFF**.

### OFF (기본 — 초보 유저)
- "오늘 뭐 했어요?"만 충족 → 선택한 블록을 **단순 칩/태그 묶음**으로만 표시
- 순서변경 UI, 온도·시간·반복·평가 입력 **모두 숨김**
- (탭 순서는 내부적으로 보존하되 표면엔 강조 안 함)

### ON (고급 — 루틴 디깅 유저)
- 선택한 블록이 **"내 루틴" 카드 스택**(세로 리스트)으로 전환
- 각 카드: `순번 · 아이콘 · 블록명 · 요약값(온도/시간/평가)` + 펼침 셰브론 + 삭제
- **카드 탭 = 인라인 펼침** → 블록 타입별 입력:
  - heat/ice(온도시설): **온도 슬라이더**(드래그, thumb 표시) + **시간 스테퍼**(−/＋) + **반복 제외** 토글
  - rest(휴식·외기욕·내기욕): 시간 스테퍼 + (선택) 휴식 질 평가
  - beyond 평가형(세신·마사지·매점·수면): **평가(1~5)** [+ 매점은 메모]
  - 그 외(아우프구스/기타): 입력 없음 또는 메모
- **순서변경**: 카드 좌측 `⋮⋮` 손잡이 **드래그**(타겟 크게). 드롭 위치 인디케이터 표시
- 블록 2개 이상이면 하단에 **반복 N세트** 스테퍼 노출

> 핵심: OFF/ON 사이를 오갈 때 **선택한 블록 집합은 유지**된다. ON에서 입력한 온도/시간도 보존.

## 4. 퀵 평가

- **트라이브 품질** (트라이브별 라벨/이모지 5단계, 1탭):
  - 사우나파=토토노우, 목욕파=수질, 찜질파=땀 만족
- **또 갈래요?** (재방문 의향 5단계, 1탭): 안가요 / 대안없음 / 근처면 / 찾아가요 / 꼭가요
- 한 줄 5버튼씩, 선택 시 강조

## 5. 더 자세히 (접힘, 선택)

- 접힌 버튼 → 펼치면: **청결도**(5단계) · **한 줄 메모**(textarea) · 추가 칩(동행 / 입장료 / 혼잡도)
- 전부 선택 입력

## 6. 저장

- 하단 풀폭 저장 버튼 `사-첵 완료`
- **활성 조건**: 블록 1개 이상 선택 + 또갈래요 선택. (나머지는 전부 선택)

- `temp:[min,max,default]` — heat/ice만. 슬라이더 범위/초기값으로 사용.

- 카테고리 색 가이드(룩에서 조정 가능): heat=따뜻한 주황, ice=파랑, rest=초록, beyond=보라.

## 상태 모델 (구현 참고)

```ts
type Mode = 'saunner' | 'bather' | 'jimi'
interface PickedBlock {
  id: string          // BLOCK_TYPES id
  temp: number | null // 온도시설만
  dur: number | null  // 분 or 초 (unit 따름)
  norepeat: boolean   // 반복 구간서 1회만
  score: number       // 0=미입력, 1~5 (평가/휴식질)
}
interface State {
  tribe: Mode
  picked: PickedBlock[]   // 탭 순서 = 동선 순서
  moreOpen: boolean       // 가로 레인 펼침
  routineDetail: boolean  // ★ 내 루틴 상세 토글 (기본 false)
  quality: number         // 0~5
  revisit: number | null  // 1~5 (저장 필수)
  clean: number           // 0~5
  detailOpen: boolean
  repeat: number          // 반복 세트
  openCard: string | null // 인라인 펼친 카드 id
}
```

핵심 동작:
- `toggleBlock(id)`: picked에 있으면 제거, 없으면 기본값으로 추가. 기본 블록 행 + 레인 칩 양쪽 순번 갱신.
- `routineDetail` 전환 시 picked 보존. OFF=칩 묶음 렌더 / ON=카드 스택 렌더.
- 저장 활성: `picked.length > 0 && revisit != null`.

## 범위 밖 (직접 입힐 것)

색/타이포/그림자/라운드/간격/아이콘 세트 등 **비주얼 룩**. 구조엔 의미 있는 클래스 훅만 남겨두면 됨.
