## 0. 전체 데이터 흐름 (변경 후)

```
[입력] log/page.tsx v6 단일폼
   └→ insertLogWithBlocks / updateLogWithBlocks
       ├→ log_blocks (정正: seq·block_type·temp·duration_sec·score·cost·memo·variant·norepeat)
       └→ logs 평탄 캐시 (buildLogCaches가 블록서 파생: 온도10·heat/ice/rest_time·평가류)

[읽기] 모든 화면 → logs-service LOG_SELECT → toLogWithPlace
   └→ ★ 여기 한 곳에서 "새 캐시 ?? 레거시(구컬럼·deep_logs)" coalesce  ← 오늘 추가
       └→ 표시면(히스토리·장소상세·카드·스토리·통계)은 전부 평탄 필드만 읽음
          (deep_log 직접 참조 0건 — 고아 페이지 log/deep 제외)
```

**왜 이 방식인가**: 폴백을 화면마다 흩뿌리지 않고 `toLogWithPlace` 1곳에 모음
→ 030(구컬럼·deep_logs DROP) 때 이 폴백 블록만 지우면 끝. 화면 코드는 더 안 건드림.
스토리 평탄화 커밋(61ad2e7)의 `dry_sauna_temp ?? sauna_temp` 패턴을 전사 표준화한 것.
