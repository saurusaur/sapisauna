## 1. 👨‍💻 개발자 — 백엔드·운영 인프라

### ① 지도 DB 2티어 적재 `P0·즉시`
정부 공공데이터(목욕장업 인허가) 전국 목욕탕 CSV — **영업중 ~5,000곳 1차 필터링 완료**.
- **전략 = 2티어 전량 노출**: 5천곳=`basic`(이름·주소·영업상태) 적재 / 우리 312곳=`rich`(온도·시설·픽). **추천·랭킹·픽 쿼리엔 rich만**, basic은 로그 쌓이면 승격.
- ⚠️ **좌표 변환**: 공공데이터 좌표 = 중부원점TM(EPSG:5174/5179) → WGS84로 변환 필수(안 하면 핀 엉뚱). 누락 행은 주소 지오코딩.
- **312곳 dedup**: 이름+주소+좌표 프록시미티 매칭 → 매칭분은 신규 X, 기존 rich에 정식명·영업상태만 보강(`scripts/katalk-*` 중복가드 재사용).
- **티어 플래그는 우리가 정의해 넘김**(`places.data_tier`) — 개발자는 적재만. source='public-data', 영업상태 보존.

### ② 애널리틱스 (PostHog) `P1·즉시`
- 도구 = **PostHog**(무료 티어), 설치+배선.
- 추적 이벤트: `signup` · `onboarding_done` · `first_log` · `log_created` · `place_saved` · `list_subscribed` · `return_visit`.
- 퍼널: 가입 → 온보딩 → 첫 기록 → 재방문. 어드민 대시보드에 KPI + Sentry 링크.

### ③ 어드민 페이지 `P1·결정 후`
스크립트로만 가능한 장소 등록·교정 + 큐레이션 관리 UI를 `/admin`으로. **베타 범위:**

| 넣음 ✅ | 뺌 ❌ (post-beta) |
|---|---|
| **B 장소·데이터**: 단건 CRUD·병합·폐업 + **2티어 운영(일괄·티어승격·basic핀 노출토글)** + 온도교정(L1후) | F 브랜드·CMS(문구/컬러/로고) — 코드상수+배포 |
| **C 큐레이션**: 어드민 직접 리스트 생성·편집 + is_featured + FEATURED 선충전 | E 유저 차단/정지/status — read-only만 |
| **A 대시보드**: KPI + 애널리틱스(②) + 5천 품질리포트 + Sentry 링크 | 양방향 메시징 · 감사로그 · G 푸시알림 |
| **D 인박스(최소)**: 인바운드 폼+큐(장소신고·피드백) — 답변은 메일 | |

- 기술: `/admin/*` + 미들웨어 가드 + **서버 재검증**. 쓰기는 service_role → **서버 route handler**(클라 service key 금지). `scripts/katalk-*` 로직 서버 API로 이식.
- 권한 = **하드코딩 ID 유지**(솔로). 온도교정 화면은 **L1 이후**.
- 상세: `docs/plans/PLAN_admin_page_scope.md` §8.

## 2. 🎨 UX 디자이너 — 화면 구조 (와이어만, 구현은 우리)

> 산출물 = **바로 구현 가능한 와이어** + **UI 5상태**(Empty/Loading/Partial/Error/Ideal) + 인터랙션 명세. (애매한 무드보드 X)

### ① 지도 페이지 와이어 — F2 + 검색흐름 `P1·즉시`
- **F2 트라이브 픽 필터**: 사우너픽/탕러픽/찜질러픽 + 영업중·24시 토글, 조합 시 실시간 카운트·핀 갱신.
- **검색 흐름 개선**: 검색 시 자동 센터링·줌 + "이 지역에서 검색"(bounds 재조회) + 자동완성. (현재는 로드된 장소만 클라 필터링·지도 미이동)
- **2티어 핀 시각 구분**: rich 312곳=강조 핀 / basic 5천곳=심플 핀. basic 핀 다수라 **클릭 시 정보·첫 기록 유도 CTA 설계가 관건.**
- ⚙️ 핀 312→5,000+ 급증 → 성능(클러스터링·마커)은 개발자와 협업.

### ② 수집·회고 화면 전체 그림 (F5 히스토리 재설계 + F4 통합) `P1·즉시`
- **"방문 = 수집" 컨셉의 전체 그림·메타포·IA를 잡아주는 게 핵심.** (이게 이번 UX 친구의 플래그십)
- 도장판(F4) + 히스토리 재설계(F5)를 **한 화면**으로 — 수집(채워짐) → 회고(탭하면 추억·좋았던/별로였던) → 공유.
- **메인 메타포는 열어둠**(유리병/앨범/뱃지 등) → 전문가가 결정. 수집 단위 = 무드 젤리 v1(🔥🛁🥚).
- 유지: 캘린더↔리스트 토글·KPI·인사이트·최근기록.
- 📄 **브리프 = `docs/plans/컨셉_사우나템_컬렉션_20260607.md`** (인스피·결정·열어둔 것 다 정리됨) — 인스피 이미지 첨부해서 전달.

### ③ F1 블록 기록 인터랙션 리뷰 `보조·즉시`
- F1 구현 거의 완료 → **블록 탭=기록(순서=루틴)·⚡빠른/✍️상세 토글** 정보구조를 디자이너 눈으로 점검·다듬기. (신규 설계 아님)

## 3. 🛠 빌더 — 게임화 기능 2개 (코드까지)

> "재밌는 걸 만들고 싶어함" → 게임화 2개 통째 빌드. 디자인은 안 맡김.

### ① F4 사우나 수집·도장판 `P1·즉시`
- 방문(체크인)=**무드 젤리 수집**(🔥🛁🥚), 월간 컬렉션이 채워짐, 완성 시 리워드+경품 추첨(브랜드 스폰서=제휴 수익화). 혼자 써도 쌓여 콜드스타트에 강함.
- **v1 = 무드 젤리(트라이브 색 그라데) + 기존 토큰** → 커스텀 아트 불필요, 즉시 빌드. **아이템(때수건·식혜·맥반석란) 확장은 나중.**
- ⚙️ **설계 팁**: 수집 데이터는 "무엇을 모았나"를 **타입/ID로 generic 저장** → 나중에 렌더만 젤리→아이템 아트로 교체(마이그레이션 X).
- 빌더 집중 = **체크인→수집 적립 · 월간 컬렉션 · 완성 판정 · 리워드 훅.**
- 화면 전체 그림은 **UX 친구가 잡음**(§2-②, 컨셉 브리프) → 빌더는 그 와이어로 빌드. F5 통합 시 회고 화면도 함께.
- 📄 컨셉 브리프: `docs/plans/컨셉_사우나템_컬렉션_20260607.md`

### ② F3 Beli식 비교 랭킹 `P2·L1 후`
- 방문지 **둘씩 비교 → 개인 점수(0~10) 자동 정렬** → "내 인생 사우나 랭킹" 누적.
- 완성된 Top은 친구 공개·비교 → **한 탭 SA-리스트화(큐레이터 전환)**.
- ⚠️ 선행: L1 스키마(점수·비교 데이터 모델).
